학습자의 대표 자료 하나를 `knowledge/원본/`에 인제스트하도록 안내한다. 워크숍 2-1 챕터 실습.

## 1. 자료 유형 확인

학습자에게 어떤 자료를 넣을지 묻는다: **PPT / 영상 / 상담 노트** 중 하나. 대표 자료 1개면 충분하다고 안내(전량 인제스트는 워크숍 후 숙제).

## 2. 유형별 처리

| 유형 | 요청 | 정리 방식 |
|---|---|---|
| **PPT** | 슬라이드 텍스트(발표자 노트 포함하면 더 좋음)를 붙여넣도록 안내 | 슬라이드 순서·구조를 살려 정리. 각 핵심 슬라이드에 번호를 남겨 나중에 출처 표기(슬라이드 N)가 가능하게 |
| **영상** | 자막 파일(.srt/.vtt)이나 전사 텍스트를 붙여넣도록 안내 | 타임스탬프(구간)를 살려 전사를 정리. 출처 표기(영상 MM:SS) 가능하게 |
| **상담 노트** | 상담 사례 메모를 붙여넣도록 안내 | 흐름을 정리하되 **눈에 띄는 실명·회사는 익명화**(A님·대기업 C 등). 꼼꼼한 점검은 2-4에서 별도 |

## 3. 파일 저장

- 저장 위치: `knowledge/원본/<알아보기 쉬운 파일명>.md` (예: `이직_가설검증_강의.md`).
- 파일명은 내용이 드러나게(코치가 나중에 "📎 근거: <파일명>"으로 인용하므로).
- 파일 상단에 자료 유형·핵심 주제를 한 줄 메타로 남긴다.

## 4. 결과 요약

저장한 파일 경로 + 무엇이 담겼는지 3줄 요약을 보여준다. 빠진 내용이 있으면 대화로 보강 가능함을 안내.

## 원칙
- 완벽한 형식을 요구하지 않는다. 대표 자료 1개로 흐름을 익히는 게 목적.
- 상담 노트는 넣는 즉시 눈에 띄는 개인정보를 익명화(전면 점검은 2-4 `/2-4-anonymize-check`).
- 파일은 반드시 `knowledge/원본/` 아래에 둔다(다른 폴더 금지).
