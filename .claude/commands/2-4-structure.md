`knowledge/원본/`의 자료를 읽어 자료 지도(`knowledge/_인덱스.md`)와 페르소나 §5(고유 관점)를 초안한다. 워크숍 2-2 챕터 실습.

## 1. 원본 읽기

`knowledge/원본/`의 모든 파일을 읽는다. 아직 자료가 하나뿐이면 그 하나로 초안(자료가 적으면 지도가 얇은 게 정상이라고 안내).

## 2. 다섯 항목 추출 → `_인덱스.md`

자료에서 반복되는 패턴을 뽑아 `knowledge/_인덱스.md`의 다섯 항목을 채운다.

| 항목 | 뽑을 것 |
|---|---|
| **1. 코칭 프레임워크** | 반복되는 모델·단계(이름 + 한 줄 요약) |
| **2. 핵심 개념·용어** | 학습자가 고유하게 쓰는 정의 |
| **3. 고유 관점·신념** | 커리어에 대해 반복하는 믿음 |
| **4. 자주 나오는 질문(FAQ)** | 상담에서 반복되는 질문 + 전형적 답변 방향 |
| **5. 자료 인벤토리** | `원본/` 파일명·유형·핵심 주제 |

각 항목에 **근거 자료**(`원본/파일명`)를 반드시 연결한다. 추론으로 채우지 말고 자료에 실제로 있는 것만.

## 3. 페르소나 §5 반영

위 3번(고유 관점·신념)을 **세 에이전트 파일**(`토론에이전트.md`·`리서치에이전트.md`·`답변에이전트.md`)의 **§5 고유 관점**에 **동일하게** 반영한다(3~5개). §5는 세 에이전트의 토론·리서치·답변이 학습자 철학과 어긋나지 않게 붙잡는 공유 나침반.

## 4. 요약 + 확인 유도

무엇을 어떻게 채웠는지 요약하고, 학습자가 표현을 바로잡을 수 있게 유도("프레임워크 이름·관점이 실제 쓰는 표현과 다르면 말씀해주세요").

## 원칙
- **추론 금지** — 자료에 없는 프레임워크·관점을 지어내지 않는다. 근거 있는 것만.
- `_인덱스.md`와 세 에이전트 §5는 **같이** 수정한다(고유 관점은 `_인덱스.md` + 세 파일 §5에 동기화).
- 학습자 피드백은 두 파일에 동시 반영.
