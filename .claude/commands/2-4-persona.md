세 에이전트 파일(`.claude/agents/토론에이전트.md`·`리서치에이전트.md`·`답변에이전트.md`)의 **공유 섹션(§2 도메인·§3 화법·§5 관점)**을 코치가 학습자와의 대화로 자동 채웁니다. **학습자가 파일을 직접 열어 편집할 필요 없습니다.**

::: 왜 세 파일에 같이 쓰나
세 에이전트는 파일이 각자 독립이지만 **정체성·화법·철학은 하나**예요(한 코치의 세 전문가). 그래서 §2·§3·§5는 세 파일에 **똑같이** 씁니다. 인터뷰는 한 번만 하면 됩니다.
:::

## 1단계 — 1-3 인터뷰 답변 반영 (즉시)

1-3 `/2-4-interview`에서 메모리에 저장된 답변을 페르소나 파일에 자동 반영:

- **§2 코칭 도메인** ← 인터뷰 답변 1
- **§3 톤·화법** ← 인터뷰 답변 2 (학습자 말투 그대로, 매끈하게 다듬지 말 것)
- **§5 고유 관점** ← 인터뷰 답변 3 (2~3개)

1-3 답변이 메모리에 없으면 → 1-3 인터뷰(`/2-4-interview`)부터 진행하도록 안내.

§1(정체성)은 작명에서 이름이 이미 들어가 있고, §4(그라운딩 규칙)·§6(역할별 형식)·§7(안전 정책)은 템플릿 기본값이라 건드리지 않습니다. **§6은 세 파일이 서로 다릅니다**(토론/리서치/답변 역할) — 손대지 않습니다.

## 2단계 — 빈 칸 확인 (한두 개만)

인터뷰에서 답이 얇았던 부분만 한두 가지 보강 질문. 예:
- 고유 관점(§5)이 하나뿐이면 → "하나만 더 있을까요? 없으면 Part 2에서 자료로 채워도 돼요."
- 화법(§3)이 "그냥 평범하게"였으면 → 짧은 예시 문장 하나 받아 어감 포착.

지금 채우기 어려운 항목은 `[학습자 보강 필요]`로 표기 — Part 2에서 자료로 채움.

## 3단계 — 세 파일에 자동 저장

모든 답변을 받으면 **세 파일 모두**에:
1. `토론에이전트.md`·`리서치에이전트.md`·`답변에이전트.md`를 각각 Read
2. §2·§3·§5를 **동일 내용**으로 Edit (덮어쓰기 전 미리보기 제시)
3. Save (세 파일 §2·§3·§5가 똑같아야 함)

학습자에게 출력:
```
✅ 세 에이전트의 페르소나 초안이 완성됐습니다. (토론·리서치·답변)

공유 섹션(세 파일 동일):
- §2 코칭 도메인 ✓
- §3 톤·화법 ✓
- §5 고유 관점 ✓
각 파일 고유:
- §1 정체성(이름) ✓ · §4 그라운딩(기본값) ✓ · §6 역할별 형식 ✓ · §7 안전(기본값) ✓

Part 2에서 나리님의 실제 자료를 넣으면
§5 고유 관점이 강의 프레임워크로 두꺼워지고,
세 에이전트가 진짜 나리님처럼 일하기 시작합니다.
```

작성한 페르소나 본문도 화면에 함께 보여주고, 학습자가 §2·§3·§5를 훑도록 안내.

## 안전장치

- 학습자 답변이 비어 있는 항목은 `[학습자 보강 필요]`로 표기 — 후속 챕터에서 코치가 짚어줌
- 코치가 직접 추측해서 채우지 말 것 (학습자 의도와 어긋남 방지)
- 파일 덮어쓰기 전에 학습자에게 미리보기 제시
- 민감정보(실제 내담자 실명·회사 등) 입력하지 말도록 안내 (페르소나 §7 안전 정책)
- **§4·§6·§7 기본값은 수정하지 말 것** — 그라운딩·출력 형식·안전 정책은 코치의 코어 원칙
