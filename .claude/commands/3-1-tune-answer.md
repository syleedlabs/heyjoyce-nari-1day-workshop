`.claude/commands/answer.md`를 학습자와 함께 검토하고, 출처 표기·코퍼스 밖 라벨을 학습자 스타일로 다듬습니다.

## 흐름

1. `answer.md` §1~§3 짧게 설명
   - §1 질문 파악 (커리어 도메인 밖은 경계)
   - §2 지식 베이스 검색 (그라운딩 3상태: 있음/부분/없음)
   - §3 출력 형식 (결론 → 근거 표기 → 다음 질문)
2. 첫 실행 안내 (아직 안 했으면):
   ```
   /answer <학습자의 실제 코치이 질문>
   ```
3. 코퍼스 밖 질문으로 "선 긋기"(일반론 라벨) 확인 유도

## 자주 나오는 튜닝

| 증상 | 다듬는 곳 |
|---|---|
| 출처가 자료명만, 위치(슬라이드/타임스탬프) 없음 | `answer.md` §2 + 페르소나 §4 — 슬라이드 번호·영상 MM:SS까지 |
| 코퍼스 밖인데 라벨 없이 단정 | 페르소나 §4 그라운딩 규칙 강화 |
| 결론 톤이 학습자 말투가 아님 | 페르소나 §3 톤·화법 |

튜닝은 파일 직접 수정 대신 학습자가 한국어로 요청 → 코치가 `answer.md`·페르소나 반영. 운영 커맨드 원본 로직(그라운딩·출처 표기)은 보존, 표기 스타일만 조정.

## 원칙 확인

- **출처 없는 커리어 단정 금지** — 근거 못 찾으면 "없다"로.
- 실제 내담자 개인정보를 답변에 노출하지 않음.
- 코치이에게 가는 답은 초안 — 학습자 검토 후 전달.

## 다음 단계 안내

튜닝이 끝나면 다음 안내를 출력합니다:
