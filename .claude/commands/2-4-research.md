**리서치 에이전트**(`.claude/agents/리서치에이전트.md`)와 그걸 부르는 `/research` 커맨드(`.claude/commands/research.md`)를 학습자와 함께 검토하고, 실제 커리어 주제 1개를 리서치합니다. `/research`를 입력하면 **메인 대화가 리서치 에이전트로 전환**된다는 점을 짚어줍니다.

## 흐름

1. `research.md` §1~§4 짧게 설명
   - §1 대상 구체화 (모호하면 1개만 되물음, 질문 남발 금지)
   - §2 조사 (WebSearch/WebFetch, 모든 수치에 출처 URL, 최신성·신뢰도)
   - §3 학습자 프레임 해석 (`knowledge/_인덱스.md` 렌즈로 의미 해석)
   - §4 출력 (요약 → 발견(출처) → 관점 해석 → 코치이에게 전할 한마디)
2. **핵심 강조** — `/research`는 검색 대체가 아니라 대행. 정보 나열을 넘어 학습자 프레임 해석까지 가야 함.
3. 첫 실행 안내:
   ```
   /research <학습자가 최근 상담에서 궁금했던 실제 주제>
   ```

## 두 정보의 라벨 구분 확인

- **리서치 사실 (웹)** → `출처: <URL>`, 못 찾으면 "확인 안 됨"
- **학습자 관점 해석 (코퍼스)** → `📎 근거: <자료명>`

웹 수치와 코퍼스 해석을 섞지 않고 라벨로 구분했는지 학습자와 확인.

## 자주 나오는 튜닝

| 원하는 것 | 다듬는 곳 |
|---|---|
| 정보 나열만, 해석 약함 | `리서치에이전트.md` §6 — "해석 단계를 더 길게, 학습자 프레임으로" |
| 출처 없는 수치 | `리서치에이전트.md` §6·§7 — "출처 없는 수치는 '확인 안 됨'" |
| 오래된 데이터를 최신처럼 | `리서치에이전트.md` §6 — "발행 연도 확인, 오래되면 표기" |

## 원칙 확인

- **출처 없는 수치 금지** — 못 찾으면 "확인 안 됨".
- 리서치 사실(웹)과 학습자 관점(코퍼스)을 라벨로 구분.

## 다음 단계 안내

리서치 1개가 끝나면 다음 안내를 출력합니다:

> "리서치 대행 완성. 이제 코치가 근거 없는 답을 스스로 걸러내게 만들 차례예요 — 큰 우와 #2. `/2-4-evaluator`를 입력하거나 2-5 챕터로 가주세요."
