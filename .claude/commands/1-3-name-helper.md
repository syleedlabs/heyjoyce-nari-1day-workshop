---
description: 코치 이름 후보 3개 제안
---

학습자의 코칭 도메인·톤 취향을 고려해 코치 이름 후보 3개를 제안합니다.

## 입력 (학습자에게 직접 묻기)

학습자에게 다음을 차례로 묻고 답을 받습니다. 한 질문 → 답 → 다음 질문 페이스로. **사전 체크리스트는 사용하지 않습니다.**

1. **코칭 도메인 한 줄** — "예: 직무 전환·이직 전략 코칭. 주 대상은 3~10년차 직장인."
2. **선호 톤** — 따뜻·경청 / 직설·명료 / 질문형 중 하나 (또는 본인 스타일 한 줄)
3. **이름 형식 선호** — 한글 / 영문 / 혼합

## 출력 형식

```
나리님의 코치 이름 후보:

1. [이름] — [추천 이유 1줄]
2. [이름] — [추천 이유 1줄]
3. [이름] — [추천 이유 1줄]

마음에 드는 이름을 고르시거나, 본인 아이디어가 있으시면 알려주세요.
```

## 작명 가이드 적용
- 짧게 (2~4자)
- 부르기 좋게 (발음 자연스럽게)
- 코치·멘토·동반자 컨셉 우선
- 본인 도메인·취향 반영 OK

## 이름 확정 시 — 페르소나 파일 이름표 달기

학습자가 이름을 결정하면:
1. 결정한 이름을 메모리에 기록 (이후 커맨드가 이 이름으로 코치를 부름)
2. 페르소나 파일 `.claude/agents/{{COACH_NAME}}.md`를 학습자가 정한 이름으로 rename (예: `제이.md`). 파일이 이미 rename돼 있으면 그대로 사용.
3. 파일 내부 §1의 `{{COACH_NAME}}` 자리에 정한 이름을 반영해도 됨. **나리님·{{COACH_DOMAIN}} placeholder는 건드리지 말 것** (도메인은 인터뷰에서 채움).

## 다음 단계 안내

이름이 정해지면 다음 안내를 출력합니다:

> "좋은 이름이에요. 이제 코치의 첫 인사를 들어보시죠. `/1-3-first-hello`를 입력해 주세요."

`/1-3-domain-interview`(코치가 학습자를 인터뷰)는 **첫 인사 다음**입니다. 작명 직후에 인터뷰로 점프시키지 마세요.
