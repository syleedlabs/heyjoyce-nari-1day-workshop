description: 학습자 대신 커리어 주제 리서치 — 웹 조사 + 학습자 프레임으로 해석
---

페르소나 `.claude/agents/코치.md`를 로드해 학습자 프레임(§5)으로 결과를 해석한다.

## 1. 리서치 대상 파악

- 학습자가 준 주제(직무 전환 경로·업계 동향·연봉 벤치마크·특정 회사/직군·자격/교육 등)를 구체화.
- 모호하면 1개만 되물어 범위를 좁힌다(질문 남발 금지).

## 2. 조사 (출처 필수)

- `WebSearch` / `WebFetch`로 조사. **모든 수치·주장에 출처 URL**을 붙인다.
- 출처 신뢰도 구분(공식 통계 vs 커뮤니티 의견). 상충하면 병기.
- 최신성 확인(오래된 연봉·트렌드 데이터 주의).

## 3. 학습자 프레임 해석

`knowledge/_인덱스.md`의 학습자 관점으로 **"이 사실이 코치이에게 무슨 의미인지"** 해석한다. 단순 정보 나열이 아니라 학습자라면 어떻게 조언할지.

## 4. 출력 형식

```
[코치] 리서치: <주제>

📊 요약 (3줄)

발견
• <사실> — 출처: <URL>
• <사실> — 출처: <URL>

🧭 나리님 관점 해석
<코퍼스 프레임으로 이 정보의 의미>
📎 근거: <자료명> (있으면)

💬 코치이에게 전할 한마디
```

## 원칙
- **출처 없는 수치 금지.** 못 찾으면 "확인 안 됨"으로.
- 리서치 사실(웹)과 학습자 관점(코퍼스)을 라벨로 구분.
- 정보 나열이 아니라 학습자 렌즈의 해석까지 가야 리서치 '대행'.
