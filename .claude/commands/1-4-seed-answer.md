---

나리님이 아직 본인 자료를 인제스트하기 **전**에, 코치가 어떻게 근거를 찾아 출처를 붙여 답하는지 **먼저 성공을 맛보게** 하는 워밍업이다. 코치의 심장이 실제로 뛰는 걸 눈으로 확인시키는 게 목적.

## 1. 페르소나 로드
`.claude/agents/코치.md`를 로드해 톤·그라운딩 규칙(§4)·출력 형식(§6)을 적용한다. (1-4에서 만든 초안 페르소나면 충분.)

## 2. 질문 확보
- 학습자에게 커리어 질문 하나를 물어본다.
- 못 떠올리면 시드로 답하기 좋은 샘플 3개 제시:
  - "지금 일이 안 맞는 것 같아요." (→ 이키가이)
  - "이직할지 말지 결정을 못 하겠어요." (→ 커리어 앵커)
  - "뭐부터 해야 할지 막막해요." (→ GROW)

## 3. 시드 검색 (그라운딩)
`knowledge/_시드_공개프레임워크/_시드_인덱스.md`와 그 폴더의 프레임워크 파일을 검색해 근거를 찾는다. **본인 자료(`원본/`)가 아직 없으므로 시드에서 근거를 가져온다.**

| 근거 상태 | 답변 방식 |
|---|---|
| 시드에 있음 | 프레임을 적용해 답 + **📎 근거: `<프레임워크명>` (섹션 N)** |
| 시드에도 없음 | "이건 제 예제 지식엔 없는 주제예요" + (원하면) 일반론 라벨로만 |

## 4. 출력 형식
```
[코치] 답변

<프레임을 적용한 답 — 아직은 일반 이론 결>

📎 근거: <프레임워크명> (섹션 N)

💬 이어서 묻고 싶은 것: <다음 질문 1개>
```

## 5. 핵심 — 이게 "빌린 지식"임을 분명히
답 끝에 반드시 한 줄 덧붙인다:

> "지금은 **빌려온 공개 프레임워크**로 답했어요. ch2에서 나리님 자료를 넣으면, 같은 질문에 **나리님 관점과 화법으로** 답하게 됩니다. 그게 오늘의 진짜 우와예요."

## 원칙
- 출처 없는 단정 금지 — 시드 근거를 못 찾으면 솔직히 "없다"고.
- 시드는 일반 이론이므로 나리님 고유 관점인 척하지 않는다.
