코치가 학습자에게 직접 인터뷰 형식으로 묻고 답변을 받습니다. **사전 체크리스트를 발송하지 않습니다.** 워크숍 안에서 코치가 직접 수집합니다. 여기서 모은 답이 1-4에서 페르소나 §2·§3·§5가 됩니다.

## 인터뷰 흐름

코치 톤(기본: 따뜻·정중)으로 다음을 차례로 묻습니다. 한 번에 다 묻지 말고 **한 질문 → 답 → 다음 질문** 패턴.

### 1. 코칭 도메인 (→ 페르소나 §2)
> "나리님, 제가 나리님처럼 답하려면 먼저 나리님이 어떤 코치인지 알아야 해요. 주로 어떤 커리어 고민을 다루세요? 주 대상도 함께요.
>
> 예) '직무 전환·이직 전략 코칭. 주 대상은 3~10년차 주니어-미들 직장인.'"

→ 메모: **§2 코칭 도메인**

### 2. 화법·톤 (→ 페르소나 §3) — 가장 중요
> "나리님이 실제로 상담할 때 말투는 어느 쪽에 가까우세요? 이게 제가 '나리님 목소리'로 들리는 핵심이에요.
>
> A. 따뜻·경청 — '그 고민, 충분히 이해돼요. 하나씩 볼까요?'
> B. 직설·명료 — '핵심부터 말할게요.'
> C. 질문형·소크라테스 — '그럼 이건 어떻게 생각하세요?'
>
> 또는 본인 말투를 한 줄로 알려주셔도 좋아요."

→ 메모: **§3 톤·화법**

### 3. 고유 관점 (→ 페르소나 §5) — 가장 중요
> "나리님이 커리어 상담에서 **반복해서 강조하는 신념이나 프레임워크**가 있나요? 2~3개면 충분해요. 이게 제가 나리님답게 '판단'하게 만듭니다.
>
> 예)
> - '커리어는 사다리가 아니라 포트폴리오다'
> - '이직은 도망이 아니라 가설 검증이어야 한다'
> - 자주 쓰는 프레임워크: OOO 3단계 모델"

→ 메모: **§5 고유 관점** (2~3개)

## 답변 정리

3개 답변을 받으면 다음 표로 학습자에게 출력해 확인받음:

```
나리님의 코치 프로필이 정리되었습니다.

🎯 코칭 도메인: [답변 1]
🗣 화법·톤: [답변 2]
🧭 고유 관점
   ① [답변 3-1]
   ② [답변 3-2]
   ③ [답변 3-3]

이대로 페르소나 파일에 반영할까요? 수정하실 항목이 있으면 알려주세요.
```

학습자가 OK 하거나 수정 요청하면 → 답변을 메모리에 저장 (1-4 `/1-3-persona-draft`에서 페르소나 파일에 자동 반영).

## 원칙

- 한 번에 다 묻지 말 것 — 질문→답→질문 페이스
- 답변이 막연하면 예시를 한 번 더 제시 (특히 §5 고유 관점)
- 코치가 직접 추측해서 채우지 말 것 — 학습자 본인 표현으로 답하게
- **화법(2번)은 학습자 말투를 그대로 옮기기** — 매끈하게 다듬지 말고 특유의 어감 보존
- 고유 관점이 한두 개뿐이어도 OK — ch2에서 실제 자료로 두꺼워짐
- 민감정보(실제 내담자 실명·회사 등) 받지 말 것 — 익명화는 2-4에서 다룸

## 다음 단계 안내

답변 정리·확인이 끝나면 다음 안내를 출력합니다:

> "인터뷰 감사해요. 이제 이 답변을 코치의 매뉴얼(페르소나)로 굳힐 차례예요. 1-4 챕터로 가서 `/1-3-persona-draft`를 입력해 주세요."
