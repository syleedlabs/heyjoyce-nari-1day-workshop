# 2-1. 내 자료 불러오기 (인제스트)

::: tip Step 5 / 13 · Part 2
이 챕터에서 — **본인의 강의 PPT·영상·상담 경험 중 대표 자료 하나를 실제로 코치의 지식 베이스에 불러옵니다.**
:::

## 지금부터가 이 워크숍의 심장

Part 1에서 코치의 **정체성**을 만들었어요. 이름을 지었고, 도메인과 화법을 인터뷰했고, 페르소나 초안까지 잡혔습니다. 그런데 지금 코치에게는 아직 **아는 게 없어요.** 성격은 있는데 지식이 비어 있는 상태예요.

Part 2에서 그 빈자리를 채웁니다. 본인이 헤이저이스 전후로 쌓아온 커리어 코칭 자산 — 강의 PPT, 영상, 상담 경험 — 을 코치가 읽을 수 있는 형태로 불러오는 일이에요. 이 과정을 **인제스트(ingest)** 라고 부릅니다.

이게 왜 이 트랙의 심장이냐면, 다른 어떤 트랙에도 없는 단계이기 때문이에요. 자비스(v4)는 바깥에서 들어오는 메일·일정을 정리해줬어요. 본인 지식이 아니라 **남이 보낸 정보**를 다뤘죠. 여기서 만드는 코치는 반대예요 — 본인 머릿속과 자료에 흩어져 있는 **지식을 꺼내 구조화**합니다.

::: tip 딜 도감과 같은 구조
흩어진 원본 데이터를 구조화된 지식으로 승격시키는 이 흐름은, 상업용 부동산 실거래를 "AX 지식 온톨로지"로 쌓아 올리는 딜 도감 프로젝트와 정확히 같은 뼈대예요. 원본 → 구조화 → 근거 있는 재사용. 본인은 지금 그 파이프라인을 커리어 도메인에 만들고 있는 겁니다.
:::

## 지식 베이스는 어떻게 생겼나

코치의 지식은 프로젝트 폴더 안 `knowledge/` 폴더에 삽니다. 구조는 단순해요.

```
knowledge/
├── README.md          ← 폴더 설명
├── _인덱스.md          ← 구조화된 지식 지도 (2-2에서 작성)
└── 원본/               ← 인제스트한 자료 (이 챕터에서 채움)
    ├── <강의명>.md      ← PPT에서 뽑은 텍스트
    ├── <영상명>.md      ← 영상 자막/전사
    └── <주제>_상담노트.md ← 정리·익명화한 상담 경험
```

이 챕터에서 채우는 건 `원본/` 폴더예요. 자료 하나하나를 텍스트 파일로 옮겨 담는 단계입니다. 다음 챕터(2-2)에서 이 원본들을 읽어 `_인덱스.md`라는 **지식 지도**로 구조화하고요.

::: warning 파일명을 알아보기 쉽게
코치는 답할 때 "📎 근거: `<자료명>`" 형태로 어느 자료에서 나왔는지 밝힙니다. 그 `<자료명>`이 바로 여기 `원본/` 폴더의 **파일명**이에요. 그러니 `강의01.md`보다 `이직_가설검증_강의.md`처럼 내용이 드러나는 이름을 지어주세요. 나중에 출처 표기가 훨씬 읽기 좋아집니다.
:::

## 자료 유형별 인제스트 방법

세 가지 대표 유형을 코치가 알아서 정리해줍니다. 본인은 원본을 손에 들고 있기만 하면 돼요.

| 자료 유형 | 본인이 준비할 것 | 코치가 하는 일 |
|---|---|---|
| **강의 PPT** | 슬라이드의 텍스트 (발표자 노트 포함하면 더 좋음) | 슬라이드 순서대로 핵심 텍스트·구조를 정리해 `원본/`에 저장 |
| **영상** | 자막 파일(.srt/.vtt)이나 전사 텍스트 | 구간(타임스탬프)을 살려 전사를 정리 |
| **상담 노트** | 상담 사례 메모 | 흐름을 정리하되 **실명·회사는 익명화**해서 저장 |

::: tip 자료가 파일이 아니어도 괜찮아요
PPT를 텍스트로 뽑는 게 번거로우면, 슬라이드 내용을 그냥 클로드 코드 대화창에 **붙여넣어도** 됩니다. 영상 자막도 마찬가지예요. 코치가 받은 내용을 정리해 `원본/` 파일로 저장합니다. 완벽한 형식이 아니어도 돼요 — 대표 자료 하나로 흐름부터 익히는 게 목적입니다.
:::

## [실습] 대표 자료 하나 불러오기

가장 자신 있는 자료, 본인 코칭을 가장 잘 보여주는 자료 **하나만** 골라주세요. 전량이 아니라 대표 1개면 충분합니다. 나머지는 워크숍이 끝나고 같은 방법으로 하나씩 늘려가면 돼요.

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/2-1-ingest-helper
```

코치가 어떤 유형의 자료인지 묻고, 유형에 맞는 방법으로 안내합니다.

- **PPT면** — 슬라이드 텍스트를 붙여넣도록 안내 → 순서대로 정리해 저장
- **영상이면** — 자막/전사를 붙여넣도록 안내 → 구간을 살려 저장
- **상담 노트면** — 메모를 붙여넣도록 안내 → 익명화하며 정리해 저장

정리가 끝나면 코치가 `knowledge/원본/` 안에 파일을 하나 만들고, 그 안에 무엇이 담겼는지 요약해서 보여줍니다.

::: warning 상담 노트라면 — 익명화는 지금 가볍게, 점검은 2-4에서
상담 노트를 넣을 때 실제 내담자 실명·회사명은 가명이나 일반명("A 대기업 5년차")으로 바꿔 넣어주세요. 지금은 코치가 정리하면서 눈에 띄는 것만 익명화합니다. 빠진 데가 없는지 꼼꼼히 훑는 **전용 점검**은 [2-4 개인정보](/part2/2-4-privacy)에서 따로 합니다. 그러니 여기서 완벽하지 않아도 괜찮아요.
:::

## [체크포인트] 원본 파일이 생겼는지 확인

실습이 끝나면 두 가지를 확인해주세요.

- **`knowledge/원본/` 폴더에 파일이 하나 생겼나요?** 파일명이 내용을 알아볼 수 있게 지어졌는지도 함께 봐주세요.
- **파일 안에 본인 자료의 핵심이 담겼나요?** 코치가 정리한 요약을 읽고, 본인이 그 자료에서 전하고 싶던 내용이 빠지지 않았는지 확인합니다. 빠진 게 있으면 대화로 "○○ 부분이 빠졌어요, 추가해주세요" 하면 코치가 파일을 보강합니다.

이 두 가지가 되면 코치의 지식 베이스에 **첫 벽돌**이 놓인 거예요. 아직은 원본 자료가 정리 없이 쌓여 있는 상태 — 다음 챕터에서 이걸 코치가 답할 때 실제로 참조하는 **지식 지도**로 구조화합니다.

::: tip 자료 인제스트 완료
본인의 대표 자료 하나가 코치의 지식 베이스에 들어왔습니다.
- `knowledge/원본/`에 자료 파일 1개 생성
- 자료 유형(PPT·영상·노트)에 맞게 정리
- 파일명·내용 확인

이제 코치에게 "무엇을 아는지"의 재료가 생겼어요. 다음은 이 재료를 코치가 꺼내 쓸 수 있게 정리할 차례입니다.
:::
