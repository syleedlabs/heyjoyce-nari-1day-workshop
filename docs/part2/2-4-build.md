# 2-4. 나만의 코치 만들기

::: tip Step 8 / 10 · ch2
이 챕터에서 — 개념을 잡았으니 이제 **본인 자료로 진짜 코치**를 만듭니다. 1-4에서 맛본 시드를 **내 지식으로 갈아끼워**, 코치가 처음으로 **내 목소리**로 답하는 순간(오늘의 큰 우와)을 만나요.
:::

## 지금부터가 이 워크숍의 심장

ch1에서 코치의 정체성을 세우고, 시드로 작동을 맛봤어요(1-4). ch2 앞부분에서 에이전트·PRD·하네스 개념도 잡았고요. 이제 그 모든 걸 **본인 자료 위에** 세웁니다.

다른 워크숍의 "나만의 에이전트 만들기"와 같은 자리예요. 다만 우리 코치는 **본인 지식에 그라운딩된** 코치라, 네 단계로 만듭니다.

| Step | 하는 일 | 커맨드 |
|---|---|---|
| **1** | 코치 PRD 확정 (2-2 개념을 실제 명세로) | `/2-4-prd` |
| **2** | 내 대표 자료 인제스트 | `/2-4-ingest` |
| **3** | 구조화 + 개인정보 익명화 | `/2-4-structure` · `/2-4-anonymize` |
| **4** | 시드→내 자료 교체 · **내 목소리 첫 답변** ⭐ | `/2-4-my-answer` |

## [Step 1] 코치 PRD 확정 (5분)

2-2에서 배운 PRD를 코치에 맞춰 확정합니다. "내 코치가 누구를·무엇을·어떤 근거로·어떤 톤·어떤 품질 기준으로 답하는가"를 한 장으로.

```
/2-4-prd
```

코치가 1-3 인터뷰·페르소나 초안을 가져와 **코치 PRD 초안**을 보여줘요. 특히 **품질 기준**에 그라운딩 철칙(📎 근거·코퍼스 밖 "일반론" 라벨·지어내기 금지)이 들어갔는지 확인해 주세요. 이게 이후 빌드의 기준선이 됩니다.

## [Step 2] 내 대표 자료 인제스트 (25분)

가장 자신 있는 자료 **하나**만 골라 코치의 지식 베이스에 불러옵니다. 전량이 아니라 대표 1개면 충분해요. 나머지는 워크숍 후 같은 방법으로 늘리면 됩니다.

```
/2-4-ingest
```

코치가 자료 유형을 묻고, 유형에 맞게 정리해 `knowledge/원본/`에 저장해요.

| 자료 유형 | 본인이 준비할 것 | 코치가 하는 일 |
|---|---|---|
| **강의 PPT** | 슬라이드 텍스트(발표자 노트 있으면 더 좋음) | 슬라이드 순서·구조 살려 정리 |
| **영상** | 자막(.srt/.vtt)·전사 텍스트 | 구간(타임스탬프) 살려 정리 |
| **상담 노트** | 상담 사례 메모 | 흐름 정리 + **실명·회사 익명화** |

::: warning 파일명을 알아보기 쉽게
코치는 `📎 근거: <자료명>`으로 출처를 밝혀요. 그 `<자료명>`이 `원본/` 폴더의 **파일명**이에요. `강의01.md`보다 `이직_가설검증_강의.md`처럼 내용이 드러나는 이름으로.
:::

## [Step 3] 구조화 + 개인정보 익명화 (25분)

원본을 코치가 실제로 찾아 쓰는 **지식 지도**로 구조화하고, 민감정보를 점검합니다.

```
/2-4-structure
```

코치가 `원본/`을 읽어 **프레임워크·핵심개념·자주 나오는 질문**을 `knowledge/_인덱스.md`로 정리해요. 이 인덱스가 코치의 지도예요 — 코치가 답할 때 여기부터 봅니다.

이어서 개인정보를 점검합니다. 상담 노트를 넣었다면 **반드시** 익명화 점검을 거치세요.

```
/2-4-anonymize
```

| 원본 | 치환 규칙 | 예시 |
|---|---|---|
| 내담자 실명 | 가명·역할로 | "김OO" → "3년차 마케터" |
| 회사명 | 업종·규모로 | "네이버" → "IT 대기업" |
| 연락처·주소 | 전량 삭제 | — |

::: warning 로컬 보관 원칙
`knowledge/원본/`의 실자료는 **공개 레포에 올리지 않아요**(`.gitignore` 처리). 자세한 규칙은 [부록 B](/appendix#b-개인정보-익명화-가이드)에.
:::

## [Step 4] 내 목소리 첫 답변 ⭐ (20분)

이제 시드를 **본인 자료로 갈아끼우고**, 코치를 실제 질문 앞에 세웁니다. 1-4에서 시드로 됐던 그 흐름이, 이번엔 **본인 강의·상담**을 근거로 돌아가요.

본인이 상담에서 자주 받는 커리어 질문 하나를 떠올려 주세요.

```
/2-4-my-answer
```

코치가 `_인덱스.md`와 `원본/`을 검색해, **본인 관점·화법으로** 답하고 출처를 붙입니다.

```
[코치] 답변

<결론 — 본인 화법으로>

📎 근거: <내 자료명> (슬라이드 N / 영상 MM:SS)

(자료 밖일 때) ⚠️ 여기서부턴 제 자료 밖 일반론입니다: <내용>

💬 이어서 묻고 싶은 것: <다음 질문 1개>
```

받은 답을 세 가지로 확인해 주세요.

- **말투가 본인 같나요?** — "분신" 여부를 가르는 감각이에요.
- **📎 근거가 진짜 본인 자료를 가리키나요?** — 실제 그 슬라이드에서 한 말이 맞는지 대조.
- **자료 밖에 라벨이 붙었나요?** — 없는 걸 본인 지식인 양 단정하지 않았는지.

마음에 안 드는 부분은 **대화로** 고쳐요. "말투를 더 따뜻하게" → 페르소나 수정 / "근거에 슬라이드 번호까지" → 커맨드 수정. 고친 뒤 같은 질문으로 다시 돌려 **비교**해 보세요. 이 **답→튜닝→재실행** 사이클이 코치를 본인답게 길들이는 가장 빠른 길이에요.

## [체크포인트] 분신을 처음 만난 순간

- **본인 자료를 인용한 답(📎 근거)을 받았나요?** 하나라도 받았다면 이 챕터의 핵심은 달성입니다.
- **"내가 말한 거랑 비슷하네"를 느꼈나요?** 완벽히 같지 않아도 그 방향이면 충분해요. 나머지는 튜닝으로 좁혀집니다.

---

::: tip 큰 우와 · 나만의 코치 탄생
1-4에서 빌린 지식으로 됐던 게, 이번엔 **본인 자료로** 실현됐어요.
- 코치 PRD → 인제스트 → 구조화·익명화 → 내 목소리 첫 답변
- 📎 근거가 **본인 강의·상담**을 가리키는 답 수신
- 답→튜닝→재실행 사이클 1회 경험

프롤로그의 약속 — "당신 코치가 당신 자료로 답합니다" — 이 실현된 순간이에요.
:::
