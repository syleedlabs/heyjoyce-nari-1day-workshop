# 2-4. 나만의 커리어 코치 만들기

::: tip Step 8 / 9 · Part 2
이 챕터에서 — 개념을 잡았으니 이제 **본인 자료로 진짜 커리어 코치**를 만듭니다. 이름을 지어 성격을 넣고, 나리님의 강의·상담을 학습시켜, 코치가 처음으로 **나리님 목소리**로 답하는 순간(오늘의 최종 성공)을 만나요.
:::

## 지금부터가 이 워크숍의 심장

1-4에서 **빌린 지식(시드)**으로 커리어 에이전트를 맛봤어요. 이제 그 자리에 **나리님의 진짜 자료**를 넣어, 남의 것이 아니라 **본인의 코치**를 만듭니다. 나나 워크숍의 "나만의 에이전트 만들기"와 같은 자리예요.

여섯 단계로 진행해요. 각 단계는 슬래시 명령어 하나로 코치를 움직입니다.

| Step | 하는 일 | 커맨드 |
|---|---|---|
| **1** | 코치 작명 + 첫 인사 | `/2-4-name-helper` · `/2-4-first-hello` |
| **2** | 코치 인터뷰 → 페르소나·PRD | `/2-4-interview` · `/2-4-persona` · `/2-4-prd` |
| **3** | 내 대표 자료 인제스트 | `/2-4-ingest` |
| **4** | 구조화 + 개인정보 익명화 | `/2-4-structure` · `/2-4-anonymize` |
| **5** | **내 목소리 첫 답변** ⭐ | `/2-4-my-answer` |
| **6** | 3모드 완성 + 근거 검수 | `/2-4-spar` · `/2-4-research` · `/2-4-evaluator` |

## [Step 1] 코치 작명 + 첫 인사 (5분)

코치에게 이름을 지어주세요. **딱 1분** — 마음에 안 들면 언제든 바꿔요.

```
/2-4-name-helper
```

이름이 정해지면 코치의 첫 인사를 들어봅니다.

```
/2-4-first-hello
```

코치가 나리님 이름을 부르며, 세 가지 모드로 무엇을 할지 첫 인사를 건네요. 이름을 갖고 나리님을 부르는 순간 애착이 생깁니다. 톤이 어색하면 대화로 바로 고치세요.

## [Step 2] 코치 인터뷰 → 페르소나·PRD (15분)

코치가 나리님을 인터뷰합니다. **어떤 코치인지** 알아야 나리님처럼 답하니까요.

```
/2-4-interview
```

세 가지를 짧게 물어요 — ① 코칭 도메인 ② 화법·톤 ③ 고유 관점 2~3개. 특히 **화법**과 **고유 관점**이 "분신"의 핵심이에요.

답이 모이면 페르소나(매뉴얼)로 굳히고, 코치 PRD로 기준선을 잡습니다.

```
/2-4-persona
/2-4-prd
```

::: warning 품질 기준에 그라운딩 철칙이 들어갔는지
코치 PRD의 품질 기준에 **📎 근거·코퍼스 밖 "일반론" 라벨·지어내기 금지**가 있는지 확인하세요. 이게 빠지면 코치가 지어냅니다.
:::

## [Step 3] 내 대표 자료 인제스트 (20분)

가장 자신 있는 자료 **하나**만 골라 코치에 불러옵니다. 전량이 아니라 대표 1개면 충분해요.

```
/2-4-ingest
```

| 자료 유형 | 준비할 것 | 코치가 하는 일 |
|---|---|---|
| **강의 PPT** | 슬라이드 텍스트 | 순서·구조 살려 `knowledge/원본/`에 저장 |
| **영상** | 자막·전사 | 구간(타임스탬프) 살려 정리 |
| **상담 노트** | 사례 메모 | 흐름 정리 + **실명·회사 익명화** |

::: warning 파일명을 알아보기 쉽게
코치는 `📎 근거: <자료명>`으로 출처를 밝혀요. 그 `<자료명>`이 파일명이에요. `강의01.md`보다 `이직_가설검증_강의.md`처럼.
:::

## [Step 4] 구조화 + 개인정보 익명화 (15분)

원본을 코치가 찾아 쓰는 **지식 지도**로 만들고, 민감정보를 점검합니다.

```
/2-4-structure
/2-4-anonymize
```

`/2-4-structure`는 프레임워크·핵심개념·자주 나오는 질문을 `knowledge/_인덱스.md`로 정리해요. `/2-4-anonymize`는 실명·회사를 가립니다.

::: warning 로컬 보관 원칙
`knowledge/원본/`·`_인덱스.md`·`HABITS.md`는 **공개 레포에 올리지 않아요**(`.gitignore`). 자세히는 [부록 B](/appendix#b-개인정보-익명화-가이드)에.
:::

## [Step 5] 내 목소리 첫 답변 ⭐ (15분)

이제 시드를 **본인 자료로 갈아끼우고** 코치를 실제 질문 앞에 세웁니다. 1-4에서 빌린 지식으로 됐던 흐름이, 이번엔 **본인 강의·상담**을 근거로 돌아가요.

```
/2-4-my-answer
```

받은 답을 세 가지로 확인하세요 — **말투가 본인 같나**, **📎 근거가 진짜 본인 자료를 가리키나**, **자료 밖에 라벨이 붙었나.** 마음에 안 드는 건 대화로 고치고("말투를 더 따뜻하게"→페르소나 / "슬라이드 번호까지"→커맨드), 같은 질문으로 다시 돌려 **비교**해 보세요. 이 **답→튜닝→재실행** 사이클이 코치를 본인답게 길들이는 가장 빠른 길이에요.

::: tip 최종 성공 · 나만의 코치 탄생
1-4에서 빌린 지식으로 됐던 게, 이번엔 **본인 자료로** 실현됐어요. 📎 근거가 **본인 강의·상담**을 가리키는 답을 받았다면 — 프롤로그의 약속이 실현된 순간이에요.
:::

## [Step 6] 3모드 완성 + 근거 검수 (15분)

`/answer`(대리응답)는 방금 완성했어요. 나머지 두 모드와, 근거를 스스로 검수하는 장치를 붙입니다.

```
/2-4-spar       # 토론 — 내 관점에 반론을 던져 논리 점검
/2-4-research   # 리서치 — 커리어 주제를 내 프레임·말투로 브리핑
/2-4-evaluator  # 근거 검수 — 답이 진짜 근거에 닿았는지 스스로 확인
```

`/2-4-evaluator`가 붙이는 **근거 검수자**(2-3 하네스의 Evaluator)는 세 가지를 봐요.

| 검수 기준 | 통과 조건 |
|---|---|
| **근거 진위** | 📎 근거가 실제 그 자료를 가리키나 |
| **코퍼스 밖 라벨** | 자료에 없는 내용에 "일반론" 라벨을 붙였나 |
| **철학 일관성** | 본인이 안 할 주장을 본인 것처럼 내지 않았나 |

하나라도 걸리면 코치는 답을 고치거나 "제 자료엔 없어요"로 물러납니다. 근거 없는 커리어 단정보다 "모른다"가 낫다 — 이게 코치를 믿고 쓸 수 있는 이유예요.

::: tip 큰 우와 — 코치가 스스로 걸러낸다
일부러 자료에 없는 걸 물어보세요. 코치가 지어내지 않고 **"일반론"으로 라벨하거나 물러나면** 성공이에요. 1-4에서 본 강사 에이전트의 "지어내기 거부"를, 이제 본인 코치가 합니다.
:::

## [체크포인트] 나만의 코치 완성

- **본인 자료를 인용한 답(📎 근거)을 받았나요?** (Step 5)
- **3모드가 본인 프레임으로 동작하고, 검수자가 근거 없는 답을 잡아내나요?** (Step 6)
- **"내가 말한 거랑 비슷하네"를 느꼈나요?** 완벽히 같지 않아도 그 방향이면 충분해요.
