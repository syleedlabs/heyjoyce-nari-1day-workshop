# 2-5. 3모드 완성 + 근거 검수

::: tip Step 9 / 10 · ch2
이 챕터에서 — `/answer`에 이어 **`/spar`(토론)·`/research`(리서치)** 를 완성하고, 코치가 **근거 없는 답을 스스로 걸러내는 검수자**(2-3 하네스의 Evaluator)를 붙입니다.
:::

## [개념] 하나의 코치, 세 가지 모드

2-4에서 대리응답(`/answer`)을 완성했어요. 코치는 같은 페르소나로 세 가지 방식으로 일합니다. 나머지 둘을 붙일 차례예요.

| 모드 | 커맨드 | 언제 쓰나 |
|---|---|---|
| **대리응답** | `/answer` | 코치이 질문에 본인이라면 할 답 (2-4에서 완성) |
| **토론** | `/spar` | 새 강의 논리 점검·어려운 케이스를 함께 뜯을 때 |
| **리서치** | `/research` | 커피챗 30분 전, 주제를 본인 프레임으로 조사 |

세 모드 모두 **같은 지식 베이스**에 그라운딩돼요. 그래서 하나를 잘 만들면 나머지도 같은 재료로 움직입니다.

## [실습 1] 토론 파트너 `/spar`

```
/2-5-build-spar
```

코치가 본인 관점에 **반론·devil's advocate**를 던지도록 만듭니다. 새 강의를 준비하기 전 논리를 점검하거나, 어려운 상담 케이스를 함께 뜯을 때 써요.

::: warning 반론은 도전이지, 본인 새 입장이 아니에요
`/spar`는 본인이 **실제로 하지 않을 주장**을 본인 입에 넣으면 안 돼요. 반론은 "코치의 도전"이지 "본인의 새 입장"이 아닙니다. 이 경계를 커맨드가 지키는지 확인해 주세요.
:::

## [실습 2] 리서치 대행 `/research`

```
/2-5-build-research
```

직무 전환 경로·업계 동향·연봉 벤치마크 같은 주제를 본인 대신 조사해, **단순 정보 나열이 아니라 본인 프레임·말투로** 브리핑하게 만듭니다. 웹 조사 결과도 본인 관점의 렌즈로 해석해요.

## [실습 3] 근거 검수자 붙이기 ⭐

이게 이 챕터의 핵심이에요. 2-3에서 배운 **Generator/Evaluator 분리**를 실제로 붙입니다. 코치가 답을 낸 뒤, **별도의 검수자**가 그 답이 진짜 근거에 닿아 있는지 확인하게 하는 거예요.

```
/2-5-add-evaluator
```

검수자는 세 가지를 봅니다.

| 검수 기준 | 통과 조건 |
|---|---|
| **근거 진위** | 📎 근거가 **실제 그 자료**를 가리키나 (가짜 출처 아님) |
| **코퍼스 밖 라벨** | 자료에 없는 내용에 "일반론" 라벨을 붙였나 |
| **철학 일관성** | 본인이 실제로 안 할 주장을 본인 것처럼 내지 않았나 |

하나라도 걸리면 코치는 답을 **스스로 고치거나 "제 자료엔 없어요"로 물러납니다.** 근거 없는 커리어 단정보다 "모른다"가 낫다 — 이게 코치를 믿고 쓸 수 있는 이유예요.

::: tip 큰 우와 — 코치가 스스로 걸러낸다
근거 없는 답을 코치가 **스스로 "일반론"으로 라벨하거나 물러나는** 순간. 이제 이 코치는 그럴듯하게 지어내지 않아요. 믿고 코치이에게 쓸 수 있는 도구가 된 거예요.
:::

## [체크포인트] 3모드 + 검수 완성

- **`/spar`·`/research`가 본인 프레임으로 동작하나요?** 반론이 도전 선을 지키고, 리서치가 본인 말투로 브리핑하는지.
- **검수자가 근거 없는 답을 잡아내나요?** 일부러 자료에 없는 걸 물어보고, 코치가 "일반론" 라벨을 붙이거나 물러나는지 확인.

---

::: tip 3모드 + 근거 검수 완료
- `/answer`·`/spar`·`/research` 세 모드가 본인 지식에 그라운딩돼 동작
- Generator/Evaluator 분리로 근거 없는 답을 스스로 걸러냄(할루방지)

이제 이 코치를 **일상에 꽂을** 차례예요.
:::
