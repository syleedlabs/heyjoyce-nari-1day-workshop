# 2-0. 빌린 지식으로 먼저 성공하기

::: tip 워밍업 · Part 2 진입 전 (약 10분)
본인 자료를 넣기 **전에**, 미리 넣어둔 예제 지식(시드)으로 코치가 어떻게 답하는지 **먼저 한 번 성공**해봅니다. 심장이 실제로 뛰는 걸 눈으로 보고 시작해요.
:::

## [개념] 왜 남의 지식으로 먼저 해보나

Part 1에서 코치의 **정체성**(이름·화법·관점)은 만들었지만, 코치의 **심장**(`knowledge/`)은 아직 비어 있어요. 지금 바로 본인 자료를 넣어도 되지만 — 그 전에 **작은 성공을 하나 먼저** 맛보는 게 좋아요. 자료 정리라는 진짜 작업에 들어가기 전에, "아, 이게 이렇게 작동하는구나"를 몸으로 아는 거죠.

그래서 이 워크북에는 **빌려온 예제 지식**이 미리 들어 있어요. 커리어 코칭에서 널리 쓰는 공개 프레임워크 세 개예요.

| 프레임워크 | 뭘 하는 도구 | 잘 맞는 질문 |
|---|---|---|
| **이키가이** | 좋아함·잘함·세상필요·보상 4원의 겹침으로 방향 찾기 | "지금 일이 안 맞는 것 같아요" |
| **GROW 모델** | 목표→현실→선택지→실행의지 순서로 대화 굴리기 | "뭐부터 해야 할지 막막해요" |
| **커리어 앵커** | "포기 못 하는 것" 8가지로 갈림길 진단 | "이직할지 말지 모르겠어요" |

이건 `knowledge/_시드_공개프레임워크/` 폴더에 들어 있어요. **본인 지식이 아니라 빌려온 배경지식**이에요. 오늘 이걸로 먼저 성공하고, 곧 본인 자료로 **갈아끼웁니다.**

::: tip 나나 워크숍의 '예제 데이터'와 같은 장치
다른 워크숍에서 예제 CSV로 먼저 결과물을 뽑아보듯, 여기선 예제 프레임워크로 코치를 먼저 돌려봅니다. 성공이 **본인 자료 준비 상태에 좌우되지 않게**, 워밍업으로 한 번 보장된 성공을 만드는 거예요.
:::

## [실습] 시드로 첫 그라운딩 답변 받기

클로드 코드 터미널에서 입력하세요.

```
/2-0-seed-answer
```

코치가 커리어 질문을 하나 물어봅니다(못 떠올리면 샘플을 제안해요). 그러면 코치가 시드 프레임워크에서 근거를 찾아 이렇게 답해요.

```
[코치] 답변

<프레임을 적용한 답>

📎 근거: 커리어 앵커 (섹션 3)

💬 이어서 묻고 싶은 것: <다음 질문 1개>

지금은 빌려온 공개 프레임워크로 답했어요. Part 2에서 본인 자료를 넣으면,
같은 질문에 본인 관점과 화법으로 답하게 됩니다.
```

여기서 **딱 하나만** 눈여겨보세요 — `📎 근거`가 붙어 있다는 것. 코치는 아무 말이나 지어내지 않고, **어느 자료에서 나온 말인지 밝히며** 답합니다. 이 습관이 코치를 믿을 수 있게 만드는 전부예요.

## [체크포인트] 코치가 근거를 붙여 답했나요

- **📎 근거가 붙은 답을 받았나요?** 아직 본인 자료가 아니어도 괜찮아요. 코치가 **출처를 밝히며 답하는 구조**를 눈으로 봤다면 이 워밍업의 목적은 달성입니다.
- **"이제 이 자리에 내 자료가 들어가면?"이 그려지나요?** 지금 시드 프레임워크 자리에 본인 강의·상담이 들어가면, 같은 답이 **본인 목소리**가 됩니다. 그게 2-3에서 만날 큰 우와예요.
