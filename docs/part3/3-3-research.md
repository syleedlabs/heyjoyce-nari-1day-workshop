# 3-3. 리서치 대행 (/research)

::: tip Step 11 / 13 · Part 3
이 챕터에서 — **코치에게 리서치를 시킵니다.** 코치가 커리어 주제를 웹으로 조사하고, **모든 수치·주장에 출처 URL**을 붙이고, 마지막엔 그 정보를 나리님 프레임으로 **해석**해줘요. 정보 나열이 아니라 "나리님이라면 이렇게 조언하겠다"까지 가야 리서치 '대행'입니다.
:::

## [개념] /research — 나 대신 조사하고, 내 렌즈로 해석

`/research`는 나리님이 직접 검색하기엔 시간이 아까운 커리어 주제를 코치가 대신 조사하는 모드예요. 직무 전환 경로, 업계 동향, 연봉 벤치마크, 특정 회사·직군, 자격·교육 과정 — 코치이 상담 전에 빠르게 배경을 잡을 때 특히 유용해요.

나리님 원래 표현으로 *"저 대신 리서치도 하는 에이전트."* 그런데 `/research`가 단순 검색과 다른 지점은 **마지막 해석 단계**예요.

| 단계 | 하는 일 |
|---|---|
| **조사** | `WebSearch`·`WebFetch`로 웹 조사, 모든 수치에 출처 URL |
| **나리님 프레임 해석** | `knowledge/_인덱스.md`의 관점으로 "이 사실이 코치이에게 무슨 의미인지" 해석 |

구글 검색은 사실을 나열해줄 뿐이에요. `/research`는 그 위에 **나리님이라면 이 정보를 어떻게 읽고 조언할지**를 얹습니다. 그래서 리서치 '대행'이지, 검색 '대체'가 아니에요.

### 두 종류의 정보를 라벨로 구분

`/research`도 그라운딩 원칙을 지켜요. 답에는 성격이 다른 두 정보가 섞이는데, 코치는 이걸 라벨로 나눕니다.

- **리서치 사실 (웹)** — 반드시 `출처: <URL>`. 못 찾으면 "확인 안 됨"으로 표기, 추정 금지.
- **나리님 관점 해석 (코퍼스)** — `📎 근거: <자료명>`으로 표기.

웹에서 온 숫자와 나리님 코퍼스에서 온 해석을 섞어버리면, 코치이가 "이게 검증된 사실인지 코치 의견인지" 헷갈려요. 라벨로 나누는 게 신뢰의 핵심입니다.

::: warning 출처 없는 수치 금지
연봉·시장 규모·채용 트렌드 같은 숫자는 **출처 URL이 붙을 때만** 답에 넣어요. 코치가 출처 없이 "요즘 평균 연봉은 X만 원"이라고 단정하면, 그건 지어낸 숫자일 수 있어요. 출처를 못 찾으면 "확인 안 됨"이 맞는 답입니다.
:::

## [실습] 실제 커리어 주제 하나 리서치하기

나리님이 최근 코치이 상담에서 실제로 궁금했던 주제 하나를 골라주세요 — 예: "데이터 분석가로 직무 전환할 때 어떤 경로가 있나", "PM 연봉이 연차별로 어떻게 되나."

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/research 마케터에서 데이터 분석가로 직무 전환하는 경로
```

코치가 다음과 같이 브리핑합니다.

```
[코치] 리서치: 마케터 → 데이터 분석가 직무 전환 경로

📊 요약 (3줄)
• 전환은 크게 부트캠프·사내이동·독학 포트폴리오 3경로
• 마케터의 강점(도메인 지식·A/B 테스트 경험)이 차별점이 됨
• 첫 전환은 '마케팅 데이터 분석' 접점 직무가 리스크가 낮음

발견
• 데이터 분석 부트캠프 수료 후 전환 사례 다수 — 출처: <URL>
• 마케팅 애널리틱스 직무 채용 공고 증가 추세 — 출처: <URL>
• 국내 데이터 분석가 연차별 연봉 벤치마크 — 출처: <URL>

🧭 나리님 관점 해석
나리님은 "전환은 리셋이 아니라 가설 검증"이라고 보시죠.
그 렌즈로 보면, 마케터의 도메인 지식을 버리는 '완전 전환'보다
접점 직무에서 데이터 역량을 검증하며 이동하는 게 이 분의
강점을 축적으로 바꾸는 길이에요.
📎 근거: 「직무전환 3단계」 강의 (슬라이드 8)

💬 코치이에게 전할 한마디
"전부 갈아엎지 말고, 지금 마케팅 데이터부터 직접 만져보세요.
거기서 재미와 적성을 검증한 다음 이동해도 늦지 않아요."
```

세 곳을 확인해보세요.

- **발견의 출처** — 수치·주장마다 `출처: URL`이 붙어 있나요? 없는 숫자가 있으면 코치에게 출처를 다시 요구하세요.
- **🧭 관점 해석** — 단순 정보 나열을 넘어 나리님 프레임으로 **의미**를 해석했나요? 여기가 핵심이에요.
- **💬 한마디** — 나리님이 코치이에게 실제로 할 법한 말인가요?

## [실습] 리서치를 내 스타일로 — /3-3-build-research

`/research`의 작동 설명서를 함께 보고 조정합니다.

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/3-3-build-research
```

코치가 `research.md`의 핵심 동작 4가지를 보여줍니다.

1. **대상 구체화** — 모호하면 딱 1개만 되물어 범위를 좁힘 (질문 남발 금지)
2. **조사** — `WebSearch`·`WebFetch`, 모든 수치에 출처 URL, 최신성·신뢰도 확인
3. **나리님 프레임 해석** — 코퍼스 렌즈로 의미 해석
4. **출력** — 요약 → 발견(출처) → 관점 해석 → 코치이에게 전할 한마디

자주 나오는 튜닝이에요.

| 원하는 것 | 다듬는 곳 |
|---|---|
| 정보 나열만 하고 해석이 약함 | `research.md` §3 — "해석 단계를 더 길게, 나리님 프레임으로" |
| 출처가 빠진 수치가 있음 | `research.md` §2 — "출처 없는 수치는 '확인 안 됨'으로" |
| 오래된 데이터를 최신처럼 씀 | `research.md` §2 — "발행 연도 확인, 오래되면 표기" |

::: info 리서치 + 대리응답의 조합
`/research`로 배경을 조사한 뒤 `/answer`로 코치이 답을 쓰면, **출처 있는 최신 정보 + 나리님 관점**이 결합된 답이 나와요. 3-5에서 이 조합을 "커피챗 30분 전 리서치 → 상담 중 대리응답"이라는 실제 습관으로 꽂습니다.
:::

---

::: tip 리서치 대행 완성
코치가 나리님 대신 조사하고, 나리님 렌즈로 해석하는 리서처가 됐어요.

- 실제 커리어 주제 1개 리서치 — 출처 URL 확인
- 웹 사실(출처)과 내 관점 해석(코퍼스)이 라벨로 구분되는지 확인
- 해석 깊이를 내 프레임으로 튜닝

이제 3모드(`/answer`·`/spar`·`/research`)가 모두 손에 들어왔어요. 그런데 아직 한 가지가 남았습니다 — **코치가 근거 없이 답할 때 그걸 누가 잡아주죠?** 다음 챕터가 v5의 신뢰 골격, 큰 우와 #2예요.
:::
