# 3-2. 토론 파트너 (/spar)

::: tip Step 10 / 13 · Part 3
이 챕터에서 — **코치를 나리님의 토론 상대로 세웁니다.** 코치가 나리님의 코칭 관점에 근거 있는 반론(devil's advocate)을 던져 생각을 벼려줘요. 단, "허수아비 반론"은 금지 — 나리님이 실제로 안 할 주장을 입에 넣지 않습니다.
:::

## [개념] /spar — 나와 토론하는 모드

`/answer`가 코치를 **바깥(코치이)을 향하게** 했다면, `/spar`는 코치를 **나리님 자신을 향하게** 합니다. 나리님이 던진 코칭 프레임워크·상담 판단·커리어 관점에 코치가 **반론을 던져** 생각을 단단하게 만들어요.

나리님 원래 표현으로 하면 *"저와 토론도 하는 에이전트."* 새 강의를 준비하거나, 어떤 상담 케이스를 어떻게 볼지 정리할 때 — **혼자 생각하면 놓치는 반대편**을 코치가 맡아줍니다.

### 스파링의 핵심 규칙 두 가지

코치가 반론을 던질 때 반드시 지키는 두 가지가 있어요. 이게 없으면 토론이 아니라 트집이 됩니다.

1. **허수아비 반론 금지** — 코치는 반론 전에 나리님 관점을 **먼저 정확히 요약**합니다. 나리님 입장을 약하게 왜곡해놓고 때리는 건 금지예요. 요약이 맞아야 반론이 유효합니다.
2. **나리님이 안 할 주장을 입에 넣지 않기** — 반론은 어디까지나 **코치의 도전**이지, 나리님의 새 입장이 아니에요. 코치가 "그럼 나리님은 이렇게 생각하시는 거네요"라며 안 하신 주장을 붙이지 않습니다.

그리고 반론마다 **근거**(코퍼스·리서치·논리)를 답니다. 근거 없는 반론은 억지니까요.

::: warning 토론의 목적은 이기는 게 아니에요
코치가 반론을 던지는 이유는 나리님을 논파하려는 게 아니라 **생각을 벼리기 위해서**예요. 좋은 반론을 만나면 나리님 관점이 더 단단해지거나, 미처 못 본 구멍이 보입니다. 코치가 이기면 실패, 나리님 생각이 날카로워지면 성공이에요.
:::

## [실습] 내 관점에 반론 받아보기

나리님이 코칭에서 자주 하는 주장 하나를 골라 던져보세요. 2-2에서 구조화한 `knowledge/_인덱스.md`의 "고유 관점" 중 하나면 좋아요 — 예: "커리어는 사다리가 아니라 포트폴리오다."

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/spar 커리어는 사다리가 아니라 포트폴리오처럼 쌓아야 한다
```

코치가 다음과 같이 스파링합니다.

```
[코치] 스파링

📌 나리님 관점 (요약):
커리어를 한 방향으로 올라가는 사다리가 아니라, 여러 경험을
병렬로 쌓는 포트폴리오로 봐야 한다.
📎 근거: 「커리어 포트폴리오」 강의 (슬라이드 5)

🥊 반론 1: 포트폴리오 서사는 '일관성 없는 이직'으로 읽힐 위험이 있어요.
   채용 담당자는 여전히 "왜 이 경로냐"의 스토리를 봅니다.
   — 근거: 나리님도 「이직 서사」 강의에서 "점을 잇는 내러티브"를
     강조하셨죠. 포트폴리오도 결국 잇는 실이 필요하다는 반론이에요.

🥊 반론 2: 초기 커리어(1~3년차)엔 오히려 한 우물이 복리로 쌓입니다.
   포트폴리오 전략이 모든 연차에 맞는 건 아닐 수 있어요.
   — 근거: 전문성 축적의 임계점 논리 (일반 커리어 이론)

🤔 되묻습니다:
포트폴리오와 일관성은 충돌하나요, 아니면 "무엇을 일관되게
가져갈지"를 고르는 문제일까요?
```

세 곳을 확인해보세요.

- **📌 관점 요약** — 코치가 나리님 입장을 **정확히** 요약했나요? 약하게 왜곡했다면 허수아비 반론이에요. 바로 지적하세요.
- **🥊 반론의 근거** — 각 반론에 근거가 붙어 있나요? 근거 없이 트집이면 튜닝 대상이에요.
- **🤔 되묻는 질문** — 나리님 생각을 한 발 더 밀어붙이나요?

## [실습] 코치를 나만의 스파링 파트너로 — /3-2-build-spar

`/spar`의 작동 설명서를 함께 보고, 나리님 토론 스타일에 맞게 조정합니다.

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/3-2-build-spar
```

코치가 `spar.md`의 핵심 동작 3가지를 보여줍니다.

1. **관점 파악** — `knowledge/_인덱스.md`에서 이 주제에 대한 나리님의 실제 관점을 찾음
2. **스파링** — 관점을 먼저 요약(허수아비 금지) → 근거 있는 반론 2~3개 → 되묻는 질문
3. **일관성 가드** — 나리님이 실제 안 할 주장을 입에 넣지 않음

자주 나오는 튜닝이에요.

| 원하는 것 | 다듬는 곳 |
|---|---|
| 반론이 너무 순함 (더 세게) | 페르소나 §5 관점 + `spar.md` §2 — "더 날카롭게 밀어붙여" |
| 반론이 억지스러움 (근거 약함) | `spar.md` §2 — "근거 없는 반론은 빼고" |
| 코치가 내 입장을 왜곡함 | 페르소나 §5에 관점을 더 명확히 적기 |

::: info 코퍼스에 관점이 없으면
아직 `knowledge/`에 그 주제에 대한 나리님 자료가 없으면, 코치는 "이 주제에 대한 나리님 자료를 못 찾았어요. 먼저 입장을 알려주시면 그 위에서 스파링할게요"라고 답해요. 지어낸 관점으로 반론하지 않기 위한 안전장치예요. 이런 주제가 자주 나오면 그게 곧 **다음에 인제스트할 자료**의 신호입니다.
:::

---

::: tip 토론 파트너 완성
코치가 나리님의 스파링 상대가 됐어요.

- 내 관점에 근거 있는 반론 2~3개 받아보기
- 허수아비 반론·억지 반론 걸러내기
- 코치를 내 토론 스타일로 튜닝

이제 코치는 나리님을 **혼자 두지 않아요**. 새 강의를 준비하든 상담 케이스를 뜯든, 반대편을 맡아줄 상대가 생겼습니다. 다음 챕터에서는 코치가 나리님 대신 **웹을 조사**하는 리서치 대행으로 갑니다.
:::
