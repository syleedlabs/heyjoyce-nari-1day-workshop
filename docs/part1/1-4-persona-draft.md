# 1-4. 코치의 매뉴얼 초안

::: tip Step 4 / 13 · Part 1
이 챕터에서 — **1-3 인터뷰 답변을 코치의 페르소나 파일에 굳혀**, 코치가 매번 같은 기준으로 나리님처럼 일하도록 매뉴얼을 완성합니다.
:::

## [개념] 왜 매뉴얼(페르소나)이 필요한가

AI는 본래 **매 대화마다 기억이 새로 시작**됩니다. 매뉴얼이 없으면 코치도 매번 처음 만난 사람처럼 일을 시작해서, 어제 알려드린 나리님의 화법·관점을 오늘 또 알려드려야 해요. 그럼 분신이 아니라 그냥 챗봇이죠.

**페르소나 파일**(`.claude/agents/코치.md`)은 코치가 일을 시작할 때마다 펼쳐 보는 매뉴얼이에요. 한 개의 마크다운 파일에 정체성·도메인·화법·그라운딩 규칙·고유 관점을 적어두면, 코치가 매번 이 파일을 읽고 **같은 기준**으로 답합니다. 어제와 오늘과 내일이 일관돼요.

특히 Part 3에서 완성할 세 모드(`/answer` `/spar` `/research`)가 **모두 이 페르소나를 읽고** 동작합니다. 한 번 잘 채워두면, 세 모드가 전부 나리님 목소리로 말하게 돼요.

::: details 페르소나 파일은 어떻게 구성되나
7개 섹션으로 구성됩니다.
- **§1 정체성** — 코치 이름, 나리님의 분신이라는 역할, 3모드
- **§2 코칭 도메인** — 어떤 커리어 고민을 다루는지
- **§3 톤·화법** — 나리님의 실제 코칭 말투
- **§4 그라운딩 규칙** — 근거 표기·자료 밖 라벨 (코치의 철칙)
- **§5 고유 관점** — 나리님이 반복하는 커리어 신념·프레임워크
- **§6 모드별 출력 형식** — /answer · /spar · /research 각각의 답변 틀
- **§7 안전 정책** — 민감정보 금지 · 전문영역 경계 · 자동 발송 없음

§1은 작명에서, §2·§3·§5는 1-3 인터뷰에서 이미 재료가 모였어요. §4·§6·§7은 코치의 기본 원칙이라 템플릿에 채워져 있습니다. 이 챕터에서 인터뷰 답변을 실제 파일에 굳힙니다.
:::

## [실습] 페르소나 초안 자동 작성

나리님이 마크다운 파일을 직접 열어 편집하실 필요는 **없습니다.** 코치가 1-3 인터뷰 답변을 가져와 파일에 직접 기록합니다. 클로드 코드 터미널에서 입력해 주세요.

```
/1-4-persona-draft
```

코치가 다음 순서로 진행합니다.

### 1단계 — 1-3 인터뷰 답변 자동 반영

1-3에서 받은 답변이 페르소나 파일에 자동으로 들어갑니다. 별도 입력은 필요 없어요.

| 페르소나 섹션 | 들어가는 내용 |
|---|---|
| **§2 코칭 도메인** | 인터뷰 1번 — 주로 다루는 커리어 고민 · 주 대상 |
| **§3 톤·화법** | 인터뷰 2번 — 나리님의 실제 코칭 말투 |
| **§5 고유 관점** | 인터뷰 3번 — 반복하는 커리어 신념·프레임워크 2~3개 |

코치가 짧게 정리해서 화면에 보여드려요. 빠진 게 없는지만 확인해 주세요.

### 2단계 — 빈 칸 확인 한두 개

인터뷰에서 답이 얇았던 부분이 있으면, 코치가 한두 가지만 더 물어봅니다. 예를 들어 고유 관점이 하나뿐이었다면 "하나만 더 있을까요?"처럼요. 지금 채우기 어려우면 `[학습자 보강 필요]`로 표시해 두고, Part 2에서 자료를 넣을 때 자연스럽게 채웁니다.

### 3단계 — 페르소나 파일 자동 저장

답변이 정리되면 코치가 페르소나 파일을 **직접 작성·저장**합니다.

저장 위치: `.claude/agents/코치.md` (작명에서 정한 이름이 파일명에 들어갑니다)

작성이 끝나면 코치가 이렇게 알려줍니다.

```
✅ 코치의 페르소나 초안이 완성됐습니다.

생성된 페르소나 파일: .claude/agents/코치.md
- §1 정체성 ✓
- §2 코칭 도메인 ✓
- §3 톤·화법 ✓
- §4 그라운딩 규칙 (기본값) ✓
- §5 고유 관점 ✓
- §6 모드별 출력 형식 (기본값) ✓
- §7 안전 정책 (기본값) ✓
```

코치가 만든 페르소나 본문도 화면에 함께 보여드려요. 한 번 쭉 훑으시면서 — 도메인이 맞는지, **화법이 본인 표현 그대로인지**, 고유 관점이 빠진 게 없는지 — 가볍게 확인해 주세요.

마음에 안 드는 부분이 있다면 코치에게 그대로 말씀하시면 됩니다. **페르소나는 대화로 수정**돼요.

예시:
- "§3 화법을 좀 더 직설적으로 바꿔주세요" → §3 자동 업데이트
- "고유 관점에 '이직은 도망이 아니라 가설 검증'도 추가해주세요" → §5 자동 업데이트

::: warning 왜 §5(고유 관점)가 아직 얇아도 괜찮은가
지금 코치의 매뉴얼은 **초안**이에요. §2·§3·§5는 나리님이 말로 알려준 것만 담겨 있습니다. 진짜 깊이는 Part 2에서 나리님의 **실제 자료**(PPT·영상·상담)를 넣고 구조화(2-2)할 때 채워져요. 그때 §5가 나리님 강의의 프레임워크로 두꺼워지고, 코치가 "내가 말한 거랑 똑같네" 싶은 답을 하기 시작합니다.
:::

| 1-3 코치 작명 + 첫 만남 | 이름 + 첫 인사 + 도메인·화법·고유 관점 인터뷰 |

다음 Part에서는 코치의 **심장을 채웁니다** — 나리님의 PPT·영상·상담을 지식 베이스로 불러와, 코치가 처음으로 나리님 자료를 인용해 답하는 순간(2-3)을 만나요. 오늘의 큰 우와가 거기 있습니다.
