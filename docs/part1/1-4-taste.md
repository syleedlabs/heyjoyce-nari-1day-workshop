# 1-4. 완성 코치 맛보기 — 시드로 첫 성공

::: tip Step 4 / 10 · ch1
이 챕터에서 — **본인 자료를 넣기 전에**, 미리 준비된 예제 지식(시드)으로 코치가 실제 커리어 질문에 **📎 근거를 붙여 답하는 걸 직접 경험**합니다. 오늘의 첫 성공이에요.
:::

## [개념] 왜 남의 지식으로 먼저 해보나

지금까지 ch1에서 코치의 **정체성**을 세웠어요. 이름을 지었고(1-3), 화법·관점을 인터뷰해 페르소나 초안까지 잡혔습니다. 그런데 코치의 **심장**(`knowledge/` 지식 베이스)은 아직 비어 있어요. 성격은 있는데 아는 게 없는 상태죠.

본인 자료를 넣는 건 ch2의 큰 작업이에요. 그 전에 — **작동하는 코치를 딱 한 번 먼저 경험**하고 갑니다. 목적지가 어떤 모습인지 눈으로 보고 나면, ch2의 자료 작업이 훨씬 수월해져요.

그래서 이 워크북엔 **빌려온 예제 지식**이 미리 들어 있어요. 커리어 코칭에서 널리 쓰는 공개 프레임워크 세 개예요.

| 프레임워크 | 뭘 하는 도구 | 잘 맞는 질문 |
|---|---|---|
| **이키가이** | 좋아함·잘함·세상필요·보상 4원의 겹침으로 방향 찾기 | "지금 일이 안 맞는 것 같아요" |
| **GROW 모델** | 목표→현실→선택지→실행의지 순서로 대화 굴리기 | "뭐부터 해야 할지 막막해요" |
| **커리어 앵커** | "포기 못 하는 것" 8가지로 갈림길 진단 | "이직할지 말지 모르겠어요" |

`knowledge/_시드_공개프레임워크/` 폴더에 들어 있어요. **본인 지식이 아니라 빌려온 배경지식**이에요. 오늘 이걸로 먼저 성공하고, ch2에서 본인 자료로 **갈아끼웁니다.**

::: tip 다른 워크숍의 '예제 데이터'와 같은 장치
다른 워크숍에서 예제 CSV로 카드뉴스를 먼저 뽑아보듯, 여기선 예제 프레임워크로 코치를 먼저 돌려봅니다. 성공이 **본인 자료 준비 상태에 좌우되지 않게**, 보장된 첫 성공을 만드는 거예요.
:::

## [실습 1] 시드로 첫 그라운딩 답변 받기

클로드 코드 터미널에 입력하세요.

```
/1-4-seed-answer
```

코치가 커리어 질문을 하나 물어봅니다(못 떠올리면 샘플을 제안해요). 그러면 시드 프레임워크에서 근거를 찾아 이렇게 답해요.

```
[코치] 답변

<프레임을 적용한 답>

📎 근거: 커리어 앵커 (섹션 3)

💬 이어서 묻고 싶은 것: <다음 질문 1개>

지금은 빌려온 공개 프레임워크로 답했어요. ch2에서 본인 자료를 넣으면,
같은 질문에 본인 관점과 화법으로 답하게 됩니다.
```

여기서 **딱 하나만** 눈여겨보세요 — `📎 근거`가 붙어 있다는 것. 코치는 아무 말이나 지어내지 않고, **어느 자료에서 나온 말인지 밝히며** 답합니다. 이 습관이 코치를 믿을 수 있게 만드는 전부예요.

## [실습 2] 3모드 살짝 맛보기

코치는 하나의 인격으로 **세 가지 모드**로 일해요. 시드 지식만으로도 세 모드가 어떻게 다른지 살짝 볼 수 있어요. 하나만 골라 돌려보세요.

| 모드 | 입력 | 지금(시드) 보이는 것 |
|---|---|---|
| **대리응답** `/answer` | "이직해도 될까요?" | 프레임 기반 답 + 📎 근거 |
| **토론** `/spar` | "제 커리어 결정에 반론 던져줘" | 관점에 도전하는 질문 |
| **리서치** `/research` | "PM 직무 전환 경로 조사해줘" | 주제 조사 브리핑 |

지금은 **빌린 지식**이라 답이 일반적이에요. ch2에서 본인 자료가 들어가면, 세 모드가 전부 **본인 목소리**로 바뀝니다. 그게 오늘의 큰 우와예요.

::: warning 아직 완벽하지 않아도 괜찮아요
시드는 일반 이론이라 코치가 "교과서처럼" 답할 수 있어요. 그게 정상이에요. 본인만의 표현·사례·관점은 ch2에서 자료를 넣을 때 입혀집니다. 지금은 **작동 방식**을 눈에 익히는 게 목적이에요.
:::

## [체크포인트] 코치가 근거를 붙여 답했나요

- **📎 근거가 붙은 답을 받았나요?** 아직 본인 자료가 아니어도 괜찮아요. 코치가 **출처를 밝히며 답하는 구조**를 봤다면 이 챕터의 목적은 달성입니다.
- **"이 자리에 내 자료가 들어가면?"이 그려지나요?** 지금 시드 프레임워크 자리에 본인 강의·상담이 들어가면, 같은 답이 **본인 목소리**가 됩니다.

빌려온 지식으로 완성 코치를 경험했습니다.

:::
