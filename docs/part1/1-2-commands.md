# 1-2. 명령어 & 네비게이션

::: tip Step 2 / 9 · Part 1
이 챕터에서 — 클로드 코드를 움직이는 **두 가지 방법**(그냥 대화 / 슬래시 명령어)을 익히고, 자주 쓰는 조작 몇 개를 손에 붙입니다.
:::

## [개념] 두 가지로 시킬 수 있어요

클로드 코드에 일을 시키는 방법은 두 가지예요.

| 방법 | 어떻게 | 언제 |
|---|---|---|
| **그냥 대화** | 평소 말로 "이 폴더 자료 정리해줘" | 대부분의 작업 — 자유롭게 부탁·수정 |
| **슬래시 명령어** | `/`로 시작하는 미리 만든 명령 | 자주 쓰는 작업을 한 단어로 재실행 |

이 워크숍의 실습은 대부분 **슬래시 명령어**로 진행돼요. 미리 지침을 담아둔 명령을 치면, 코치가 그대로 움직입니다. 명령이 뭘 하는지는 매번 챕터가 안내하니 외울 필요 없어요.

## [개념] 두 종류의 슬래시 명령어

| 종류 | 예 | 성격 |
|---|---|---|
| **실습 진행 명령** | `/1-4-seed-answer` | 챕터를 따라가며 한 단계씩 (번호형, 워크숍용) |
| **운영 명령** | `/answer` `/spar` `/research` | 워크숍 후에도 평생 쓰는 코치의 3모드 (단어형) |

번호형은 오늘 실습을 위한 안내판이고, 단어형은 나리님이 워크숍 뒤에 실제로 쓰게 될 도구예요.

## [개념] 자주 쓰는 조작

몇 개만 손에 붙이면 됩니다.

- **취소·중단** — 코치가 엉뚱하게 가면 `Esc`로 멈추고 다시 말하면 돼요.
- **대화로 수정** — 파일을 직접 열 필요 없어요. "방금 그거 이렇게 바꿔줘"라고 말하면 코치가 고칩니다.
- **파일 언급** — "이 폴더의 ○○ 파일 읽어줘"처럼 자연어로 가리키면 됩니다.

::: tip 완벽하게 치지 않아도 돼요
명령어를 잘못 쳤거나 결과가 이상해도 괜찮아요. 그냥 말로 "다시 해줘", "이건 취소" 하면 됩니다. 클로드 코드는 **대화로 되돌릴 수 있는** 도구예요.
:::

## [실습] 한 번 말 걸어보기

거창한 것 말고, 코치에게 가볍게 말 한 번 걸어보세요. 예를 들어:

> "지금 이 폴더에 어떤 파일들이 있는지 보여줘."

코치가 폴더를 훑어 목록을 보여줄 거예요. 이게 챗봇과 다른 지점이에요 — 코치는 화면 속 대화가 아니라 **나리님의 실제 폴더**를 들여다봅니다.

## [체크포인트] 여기까지 잡으셨다면

- **대화 / 슬래시 명령어** 두 방법이 있다는 것
- 번호형(실습) vs 단어형(운영, `/answer`·`/spar`·`/research`) 명령의 차이
- 막히면 `Esc` + 대화로 수정하면 된다는 것

이제 조작법을 익혔으니, 실제로 **자동화가 어떤 건지** 눈으로 볼 차례예요.
