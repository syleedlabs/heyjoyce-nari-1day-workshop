# 1-4. 자동화 맛보기 : 미니 코치 첫 실행

::: tip Step 4 / 9 · Part 1
1-3에서 강사 에이전트를 봤으니, 이제 나리님의 커리어 코치를 직접 돌려봅니다 — 나리님이 첫째로 바란 **"저 대신 리서치"**부터요. Part 2에서 제대로 만들 코치의 **미니 버전**이에요.
:::

## 오늘 만들어볼 결과물, 먼저 봅니다

나리님이 코치에게 커리어 주제를 주면, 코치가 **대신 조사해서 브리핑**해줘요. 오늘 받아볼 결과물이에요.

```
[코치] 리서치 브리프 — "마케터 → PM 직무 전환, 어디서 시작?"

· PM 전환은 '새로 취업'보다 지금 자리에서 제품 의사결정 경험을 쌓는 게 빠른 경로.
· 핵심 역량은 코딩이 아니라 문제 정의·우선순위·데이터 해석 — 마케팅 강점과 겹침.
· 첫 스텝: 지금 팀의 제품 결정 1건에 자원해 "왜 이걸 만드는가"를 정의해보기.

📎 근거: 웹 리서치(직무 전환 경로 자료) + 커리어 앵커(섹션 3) 렌즈로 해석
💬 이어서 묻고 싶은 것: 지금 일에서 '제품 결정'에 가장 가까운 업무는 무엇인가요?
```

핵심은 두 가지예요 — **조사한 걸 커리어 프레임으로 해석**했다는 것, 그리고 **📎 출처**가 붙었다는 것. 1-3에서 강사 에이전트가 리서치→제작으로 흘렀듯, 나리님의 코치는 **리서치→브리프**로 일해요.

## [개념] 왜 리서치부터 맛볼까?

나리님이 코치에게 바란 첫 문장이 **"저 대신 리서치도 하고"**였어요. 코치가 커피챗·상담 전에 주제를 대신 조사해주는 것 — 가장 자주, 가장 먼저 쓰게 될 기능이죠. 그래서 1-4에서 이걸 먼저 맛봅니다.

::: tip 나머지 도구는 Part 2에서 직접
나리님이 원한 **토론(반론 던지기)·대리응답(나 대신 답)**은 Part 2(2-4)에서 나리님이 **직접 코치에 더합니다.** 오늘은 그중 리서치 하나로 "아, 이렇게 일하는구나"를 먼저 익혀요.
:::

## [개념] 이 브리프가 나오는 파이프라인 + 도메인

### 파이프라인 흐름

```
커리어 주제  →  조사(웹·자료)  →  커리어 프레임으로 해석  →  브리프 + 📎 출처
```

정보를 나열만 하는 게 아니라, **커리어 관점으로 해석**해서 "그래서 뭘 하면 되는지"까지 줘요.

### 해석 렌즈 — 커리어 코칭 도메인

아직 나리님 자료를 넣기 전이라, 코치는 커리어 코칭에서 널리 쓰는 **공개 프레임워크 3개**를 조사 결과의 **해석 렌즈**로 씁니다.

| 프레임워크 | 렌즈로 보면 |
|---|---|
| **이키가이** | 좋아함·잘함·세상필요·보상 4원으로 방향 잡기 |
| **GROW 모델** | 목표→현실→선택지→실행 순서로 정리 |
| **커리어 앵커** | "포기 못 하는 것"으로 선택 진단 |

::: warning 아직 '빌린' 렌즈예요
지금은 일반 리서치 + 빌린 프레임이라 브리프가 다소 교과서적일 수 있어요. Part 2에서 나리님의 실제 강의·상담을 넣으면, **나리님 관점·말투로** 브리핑하게 됩니다.
:::

## [실습 1] 대화형으로 리서치 시켜보기 (첫 성공)

먼저 **평소 말하듯** 부탁해 봅니다. 조사하고 싶은 커리어 주제를 하나 정해 이렇게 입력해 보세요.

```
이 주제를 조사해서 커리어 관점으로 브리핑해줘. 어디서 나온 정보인지 📎 출처도 붙여줘.
주제: "마케터에서 PM으로 직무 전환하려면"
```

코치가 조사한 뒤, 위에서 본 것처럼 **핵심 3줄 + 📎 근거 + 다음 질문**으로 브리핑해요. 방금 그 결과물이 나리님 손에서 나온 거예요 — **첫 성공**이에요. 🎉

::: tip 그런데… 매번 이렇게 길게?
잘 나왔지만, 이 부탁을 매번 타이핑하긴 번거롭죠. 그래서 다음 실습이 있어요.
:::

같은 지침을 **미리 담아둔 슬래시 커맨드**가 있어요. 한 단어면 됩니다.

```

```

코치가 조사할 주제를 물어보고(못 떠올리면 샘플 제안), **실습 1과 똑같이** 브리핑해요. 강사가 `run_crawl.sh` 한 줄로 크롤을 돌렸던 것과 같은 원리 — **자주 하는 일을 커맨드로 묶으면** 다음부턴 한 단어예요.
