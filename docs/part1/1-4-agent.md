# 1-4. 커리어 에이전트 만들기 (자동화 맛보기)

::: tip Step 4 / 9 · Part 1
이 챕터에서 — 먼저 **강사의 에이전트가 리서치부터 콘텐츠 제작까지 자동으로 해내는 걸** 보고, "나도 이 정도 되겠구나"를 느낀 뒤, 나리님의 **커리어 에이전트를 처음 돌려** 첫 성공을 경험합니다.
:::

## [보기] 강사의 에이전트는 이렇게 일합니다

말로만 "자동화가 된다"고 하면 감이 안 오죠. 그래서 먼저 **강사가 실제로 쓰는 콘텐츠 제작 에이전트**를 보여드릴게요. 강사가 시킨 건 딱 한 줄이에요.

> "이 주제 리서치해서 콘텐츠 만들어줘."

그다음 에이전트가 **혼자서** 이걸 다 합니다.

| 단계 | 에이전트가 한 일 |
|---|---|
| **① 리서치** | 뉴스를 훑어 관련 후보를 모으고, 쓸 만한 것만 선별 |
| **② 팩트체크** | 기사 원문을 열어 수치·주체를 교차검증 — **보도 수치의 오류를 스스로 잡아냄** ("보도가 6,862억? 원문 확인하니 875억이네요") |
| **③ 출처 표기** | 어느 매체가 어느 수치의 근거인지 카드에 명기 (땅집고=875억 / 서울경제=매입가) |
| **④ 지어내기 거부** | 공개된 이미지가 없자 **"없는 걸 지어내지 않겠다"**며 대체안을 제안 |
| **⑤ 콘텐츠 산출** | 검증된 사실로 카드 콘텐츠를 완성 |

사람이 반나절 걸릴 일(자료 조사 → 사실 확인 → 출처 정리 → 제작)을, **지시 한 줄**로 돌린 거예요.

::: tip 여기서 느끼셨으면 하는 것
"나도 이 정도 수준으로 자동화가 가능하겠구나."

맞아요. 오늘 나리님이 만드는 커리어 에이전트도 **똑같은 규율**로 일해요 — 리서치하고, 사실을 확인하고, 출처를 밝히고, 없는 건 지어내지 않습니다. 도메인만 부동산 콘텐츠에서 **커리어 코칭**으로 바뀔 뿐이에요.
:::

## [개념] 방금 본 자동화의 정체

강사의 에이전트가 특별한 마법을 부린 게 아니에요. 핵심은 두 가지예요.

- **리서치 + 제작을 지시 한 줄로** — 사람이 단계마다 붙어 있지 않아도, 에이전트가 알아서 조사하고 만든다.
- **정직함이 내장돼 있음** — 근거를 대고, 틀린 걸 스스로 잡고, 없는 건 "없다"고 한다. (똑똑함보다 이게 더 중요해요. 특히 커리어 조언은 틀리면 사람 인생에 영향을 주니까요.)

오늘 나리님은 이 원리를 **본인 커리어 도메인**에 적용합니다. 지금 이 챕터에서는 그 커리어 에이전트를 **처음 한 번 돌려보는 것**으로 시작해요. (본인 자료로 제대로 만드는 건 Part 2에서.)

## [실습] 커리어 에이전트 첫 실행

아직 나리님 자료를 넣기 전이에요. 그래서 이 워크북엔 **미리 준비된 예제 지식(시드)**이 들어 있어요 — 커리어 코칭에서 널리 쓰는 공개 프레임워크(이키가이·GROW·커리어 앵커) 몇 개예요. 이걸로 커리어 에이전트가 어떻게 답하는지 먼저 봅니다. (나나 워크숍에서 예제 데이터로 카드뉴스를 먼저 뽑아보는 것과 같은 장치예요.)

클로드 코드 터미널에 입력하세요.

```
/1-4-seed-answer
```

에이전트가 커리어 질문을 하나 물어봅니다(못 떠올리면 샘플을 제안해요). 그러면 시드에서 근거를 찾아 이렇게 답해요.

```
[코치] 답변

<프레임을 적용한 답>

📎 근거: 커리어 앵커 (섹션 3)

💬 이어서 묻고 싶은 것: <다음 질문 1개>

지금은 빌려온 공개 프레임워크로 답했어요. Part 2에서 본인 자료를 넣으면,
같은 질문에 본인 관점과 화법으로 답하게 됩니다.
```

**딱 하나만** 눈여겨보세요 — `📎 근거`가 붙어 있다는 것. 강사의 콘텐츠 에이전트가 출처를 명기하던 것과 똑같아요. 커리어 에이전트도 **아무 말이나 지어내지 않고, 어디서 나온 말인지 밝히며** 답합니다.

::: warning 아직 완벽하지 않아도 괜찮아요
시드는 일반 이론이라 답이 "교과서처럼" 나올 수 있어요. 정상이에요. 나리님만의 표현·사례·관점은 Part 2에서 본인 자료를 넣을 때 입혀집니다. 지금은 **작동 방식**을 눈에 익히는 게 목적이에요.
:::

## [체크포인트] 첫 성공을 확인해요

- **강사 에이전트가 리서치→제작을 자동으로 하는 걸 봤나요?** 그리고 "나도 되겠다" 싶었나요?
- **커리어 에이전트에게서 📎 근거가 붙은 답을 받았나요?** 아직 본인 자료가 아니어도, 출처를 밝히며 답하는 구조를 봤다면 이 챕터의 목적은 달성입니다.

---

::: tip Part 1 완료 · 첫 성공
- 강사의 콘텐츠 에이전트가 **지시 한 줄로 리서치+제작**하는 걸 봤어요
- 나리님의 커리어 에이전트를 처음 돌려 **📎 근거 붙은 답**을 받았어요

Part 1에서 클로드 코드의 기초와 "자동화가 이 정도 된다"는 감을 잡았어요. Part 2에서는 이게 **왜/어떻게** 되는지 개념을 잡고, **본인 자료로 진짜 커리어 코치**를 만듭니다.
:::
