# 1-3. 명령어 & 코치 작명

::: tip Step 3 / 10 · ch1
이 챕터에서 — **슬래시 명령어로 코치를 움직이는 법**을 익히고, 코치에게 **이름**을 지어주고, 코치가 나리님을 인터뷰해 **페르소나 초안**까지 만듭니다.
:::

## [개념] 슬래시 명령어 — 코치를 부르는 법

이 워크숍의 실습은 대부분 **슬래시 명령어**로 진행돼요. 터미널에 `/`로 시작하는 명령을 입력하면, 미리 만들어둔 지침대로 코치가 움직입니다.

| 형태 | 예 | 언제 |
|---|---|---|
| **실습 진행 명령** | `/1-3-name-helper` | 챕터를 따라가며 한 단계씩 (번호형) |
| **운영 명령** | `/answer` `/spar` `/research` | 워크숍 후에도 평생 쓰는 코치의 3모드 (단어형) |

지금은 실습 명령을 하나씩 따라 치면 돼요. 명령이 뭘 하는지는 매번 챕터가 안내합니다. 오타가 나거나 결과가 이상하면, 그냥 코치에게 **말로** 고쳐달라고 하면 됩니다 — 클로드 코드는 대화로 움직여요.

## [실습] 작명 (1분)

코치에게 이름을 지어주세요. **딱 1분**입니다. 길게 고민하지 마세요 — 마음에 안 들면 나중에 언제든 바꿀 수 있어요.

작명 가이드:
- **짧게 (2~4자)** — 매일 부를 이름이라 호흡이 길면 부담
- **부르기 좋게** — 발음이 자연스러운지
- **본인 취향 반영 OK** — 영문/한글, 코치·멘토·동반자 컨셉, 좋아하는 어감

혼자 정하기 막막하시면 코치가 후보를 제안하게 할 수 있어요.

```
/1-3-name-helper
```

**기대 결과** — 나리님의 코칭 도메인·톤 취향에 맞춘 이름 후보 3개. 마음에 드는 걸 고르시거나 본인 아이디어로 정하셔도 됩니다. 이름이 정해지면 코치의 페르소나 파일(`.claude/agents/코치.md`)이 그 이름표를 답니다.

## [실습] 코치의 첫 한 마디 ⭐

::: tip 첫 만남
코치가 살아 움직이기 시작하는 순간입니다.

```
/1-3-first-hello
```
:::

**기대 결과** — 코치가 나리님 이름을 부르며, 세 가지 모드로 무엇을 할 수 있는지 첫 인사를 건넵니다.

> "나리님, 안녕하세요. 저는 **코치**입니다.
>
> 오늘부터 저는 나리님의 지식을 학습한 커리어 코치 분신이 됩니다. 세 가지로 함께할게요 — 나리님과 **토론**하고(`/spar`), 나리님 대신 **리서치**하고(`/research`), 코치이 질문에 나리님이라면 할 **답**을 대신 합니다(`/answer`).
>
> 아직 제 안은 비어 있어요. 나리님의 자료로 저를 채우는 게 오늘 할 일입니다. 잘 부탁드려요."

톤이 격식 있거나 호칭이 어색하면 그대로 코치에게 말씀해 보세요. **대화로 수정**됩니다.

## [실습] 코치 인터뷰 — 나를 알려주기

이제 반대로, 코치가 나리님을 인터뷰합니다. 나리님이 **어떤 코치인지**를 알아야 나리님처럼 답할 수 있으니까요.

```
/1-3-domain-interview
```

코치가 세 가지를 차례로 묻습니다. 짧게, 본인 표현 그대로 답하시면 돼요.

| # | 코치가 묻는 것 | 무엇을 답하나 |
|---|---|---|
| 1 | **코칭 도메인** | 어떤 커리어 고민을 주로 다루시는지 · 주 대상 (예: "직무 전환·이직 전략, 3~10년차") |
| 2 | **화법·톤** | 나리님의 실제 코칭 말투 (따뜻·경청 / 직설·명료 / 질문형, 또는 본인 스타일 한 줄) |
| 3 | **고유 관점 2~3개** | 반복해서 강조하는 커리어 신념·프레임워크 (예: "커리어는 사다리가 아니라 포트폴리오다") |

가장 중요한 건 **2번과 3번**이에요. **화법**은 코치가 나리님 목소리로 들리게 하고, **고유 관점**은 코치가 나리님답게 판단하게 만듭니다.

지금 답이 얇아도 괜찮아요 — ch2에서 나리님 자료를 넣으면 더 풍부해지고, 대화로 계속 다듬습니다.

## [실습] 페르소나 초안 굳히기

인터뷰 답변을 코치의 **매뉴얼(페르소나 파일)**로 굳힙니다. 나리님이 파일을 직접 열 필요는 없어요 — 코치가 인터뷰 답변을 가져와 직접 기록합니다.

```
/1-3-persona-draft
```

페르소나 파일(`.claude/agents/코치.md`)에 다음이 들어가요.

| 섹션 | 들어가는 내용 |
|---|---|
| §2 코칭 도메인 | 인터뷰 1번 |
| §3 톤·화법 | 인터뷰 2번 |
| §5 고유 관점 | 인터뷰 3번 |
| §4·§6·§7 | 그라운딩 규칙·출력 형식·안전 정책 (기본값) |

코치가 작성한 페르소나를 화면에 보여줘요. 도메인이 맞는지, **화법이 본인 표현 그대로인지** 가볍게 확인해 주세요. 마음에 안 들면 대화로 고칩니다.

::: warning 지금은 초안이에요
§2·§3·§5는 나리님이 말로 알려준 것만 담겨 있어요. 진짜 깊이는 ch2에서 **실제 자료**(PPT·영상·상담)를 넣을 때 채워집니다. 그때 코치가 "내가 말한 거랑 똑같네" 싶은 답을 하기 시작해요.
:::

::: tip [체크포인트] 여기까지 하셨다면
- 코치에게 **이름**이 생겼다 (`/1-3-name-helper`)
- 코치가 내 이름을 부르며 **3모드로 첫 인사**를 했다 (`/1-3-first-hello`)
- 코치에게 **도메인·화법·고유 관점**을 알려주고, **페르소나 초안**을 굳혔다
:::
