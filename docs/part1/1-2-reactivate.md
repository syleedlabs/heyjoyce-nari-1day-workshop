# 1-2. 작업실 재점검 + 안전 약속

::: tip Step 2 / 9 · Part 1
이 챕터에서 — **이미 깔려 있는 클로드 코드를 다시 깨우고**, "왜 그동안 못 썼는지"를 진단한 뒤, 코치와 함께 지킬 안전 약속을 잡아봅시다.
:::

## [개념] "설치는 했는데 못 쓰고 있어요"

나리님, 사실 클로드 코드는 이미 노트북에 깔려 있으시죠. 그런데 거의 안 쓰고 계실 거예요. 괜찮습니다 — **아주 흔한 일**이고, 오늘 그걸 풀려고 모였어요.

이 챕터는 "설치"가 아니라 **재활성화**입니다. 대부분 진짜 이유는 설치가 안 돼서가 아니라 이거예요.

> **"뭘 시켜야 할지 몰라서, 빈 화면 앞에서 그냥 껐다."**

빈 커서만 깜빡이는 터미널은 누구에게나 막막합니다. 그런데 오늘은 다릅니다 — 나리님에겐 **시킬 일(3모드)**과 **재료(내 자료)**가 이미 있어요. 빈 화면 공포를 없애는 게 이 챕터의 목적입니다.

## [개념] 클로드 코드 = 코치의 작업실

클로드 코드는 코치가 살고 일하는 **작업실**이에요. 이 작업실 안에서 코치는

- `.claude/agents/코치.md` 페르소나 파일을 읽어 "자기가 누구인지" 기억하고,
- `/answer` `/spar` `/research` 같은 슬래시 커맨드로 일을 실행하고,
- `knowledge/` 폴더의 나리님 자료를 뒤져 근거를 찾습니다.

나리님은 파일을 직접 편집하실 필요가 없어요. **대화와 슬래시 커맨드만으로** 코치를 움직입니다.

## [실습] 작업실 재점검 + "왜 안 썼나" 진단

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력해 주세요.

```
/1-2-reactivate-check
```

**기대 결과** — 두 가지를 함께 확인합니다.

1. **작업실 점검** — 클로드 코드 버전, OS/셸, `.claude/` 폴더 구조가 제대로 있는지 ✅/❌로 표시
2. **재활성화 진단** — "그동안 왜 못 쓰셨어요?"를 코치가 짧게 물어봅니다. 답을 들으면 그 막힘에 맞는 **첫 실마리 한 줄**을 제안해요

진단 질문에는 편하게 답하시면 됩니다 — "뭘 시킬지 몰랐다", "터미널이 무섭다", "한번 해봤는데 감이 안 왔다" 무엇이든요. 정답이 없는 질문이에요.

::: details 클로드 코드가 아직 설치 안 돼 있다면
드물지만 설치가 안 돼 있으면 `/1-2-reactivate-check`가 그 사실을 알려줍니다. 그때는 [클로드 코드 공식 안내](https://docs.claude.com/claude-code)를 따라 설치(Pro/Max 구독 필요)하신 뒤 커맨드를 다시 실행해 주세요. 이미 깔려 있으면 30초면 끝납니다.
:::

## [개념] 코치와의 안전 약속

이제부터 코치는 나리님의 **강의 자료와 상담 경험**을 학습합니다. 상담 노트에는 실제 사람 이야기가 담겨 있으니, 시작 전에 안전 약속 네 가지를 씨앗으로 심어둘게요. (각각의 상세 실습은 뒤에서 다룹니다.)

**① 지식 베이스는 로컬 보관**
나리님 자료가 모이는 `knowledge/` 폴더는 **나리님 컴퓨터에만** 있습니다. 공개 GitHub 레포에는 워크북과 코치 구조만 올라가고, 실제 자료는 `.gitignore`로 배포에서 제외돼요. → 상세: [2-4](/part2/2-4-build)

**② 상담 노트는 익명화 후 인제스트**
상담 사례를 넣을 땐 내담자 실명·회사·민감정보를 **가명/일반화**한 뒤에 넣습니다. 코치가 실제 개인정보를 학습하지 않도록요. → 상세: [2-4](/part2/2-4-build) · [부록](/appendix)

**③ 출처 표기가 곧 신뢰**
코치의 모든 답에는 `📎 근거: <자료명>`이 붙습니다. 어디서 나온 말인지 추적되니, 나리님도 코치이도 그 답을 검증할 수 있어요. 근거 없는 답은 "일반론"으로 라벨됩니다. → 상세: [2-4](/part2/2-4-build)

**④ 코치는 자동으로 발송하지 않음**
코치이에게 갈 답변은 **초안까지만** 만듭니다. 실제 전달은 언제나 나리님이 검토하고 직접 하세요. 코치는 정리·초안·추천을 하고, 결정은 나리님 몫입니다.

> 💡 이 네 가지는 코치를 **안심하고** 쓰기 위한 최소 약속이에요. "로컬 보관·익명화·출처 표기·초안까지" — 이 리듬만 기억하시면 됩니다.

::: tip [체크포인트] 여기까지 하셨다면
- `/1-2-reactivate-check` 실행 → 작업실 ✅ 확인 + 내가 왜 못 썼는지 한 번 말해봄
- 안전 약속 4가지를 눈으로 훑음 (로컬 보관 · 익명화 · 출처 표기 · 초안까지)

빈 화면 공포가 조금 가셨나요? 이제 코치에게 이름을 지어줄 시간이에요.
:::
