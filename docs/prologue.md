# 0. 프롤로그 : 내 지식이 말을 한다면

::: tip 5분 · 워크숍 시작 직전
오늘 4시간 동안 **무엇을 만드는지** 먼저 보고 시작합니다.
:::

나리님님이 지금까지 쌓아온 것을 떠올려 봅시다. 강의 PPT 수십 장, 촬영해둔 영상, 그리고 수없이 반복한 커리어 상담. 그 안에는 나리님님만의 프레임워크와 관점이 이미 다 들어 있어요.

그런데 그 지식은 대부분 **나리님님 머릿속과 파일 여기저기에 흩어져** 있습니다. 나리님님이 자리에 없으면 아무도 꺼내 쓰지 못하고, 같은 질문을 받을 때마다 매번 처음부터 다시 설명하죠.

이런 상상을 해볼게요. **내 머릿속 지식이 밖으로 걸어 나와, 내 목소리로, 내 자료를 근거로 나 대신 말한다면?** 챗봇처럼 아무 말이나 지어내는 게 아니라 — 오직 나리님님이 쌓은 자료에만 근거해서요. 이야기 속에서만 있던 '제2의 나', 분신(分身). 오늘 만드는 건 딱 그거예요. **나리님님 지식을 학습한 커리어 코치 분신.**

## 잠깐, 실물을 먼저 보시죠

말로만 하면 감이 안 오실 거예요. 그래서 먼저 **강사(이수연)의 코치**가 실제 커리어 질문에 어떻게 답하는지 보여드립니다.

누군가 이렇게 물었다고 해볼게요.

> "AI 때문에 제 직무가 사라질까 봐 불안해요. 지금이라도 개발을 배워야 할까요?"

강사의 코치가 답합니다 — **강사가 만든 강의 슬라이드를 근거로 인용하면서요.**

```
[코치] 답변

결론부터요. 지금 급하게 개발 언어부터 배우실 필요는 없어요.
먼저 지금 직무에서 "AI로 대체되지 않는 판단"이 무엇인지부터 찾는 게 순서예요.
도구는 계속 바뀌지만, 무엇을 풀 문제로 정의할지는 여전히 사람이 하거든요.

📎 근거: 「AX 전환 실전」 슬라이드 8 — "도구는 바뀌어도 문제정의는 사람이 한다"

⚠️ 여기서부턴 제 자료 밖 일반론입니다: 어떤 언어를 배울지 같은 구체적 추천은
   제 강의 자료엔 없어요. 필요하시면 일반론으로만 짚어드릴게요.

💬 이어서 묻고 싶은 것: 지금 직무에서 "이건 나여야만 한다" 싶은 판단이 있으세요?
```

보통 챗봇 답변과 뭐가 다른지 눈치채셨나요? 세 가지예요.

- **결론이 강사의 관점**입니다 — "개발부터 배워라"가 아니라 강사가 늘 강조하던 "문제정의는 사람이 한다"는 프레임이에요.
- **근거에 출처가 붙어 있어요** (`📎 근거: 「AX 전환 실전」 슬라이드 8`). 어느 자료의 어디에서 나온 말인지 추적됩니다.
- **자료 밖은 솔직히 선을 긋습니다** (`⚠️ 여기서부턴 일반론`). 지어내지 않아요.

::: tip 강사 데모 (호스트용)
강사가 본인 코치로 라이브 데모를 보여드립니다. 자료가 준비 안 됐을 때의 백업으로, 클로드 코드 터미널에서 아래 커맨드로 같은 샘플을 출력할 수 있어요.

```
/0-prologue-demo
```
:::

## 코치가 일하는 세 가지 방식

이 코치는 하나의 인격으로, 세 가지 모드로 일합니다. 나리님님이 원래 하시던 일을 그대로 옮긴 거예요.

| 모드 | 커맨드 | 하는 일 |
|---|---|---|
| **토론** | `/spar` | 나리님님 관점에 반론을 던져 생각을 벼려줍니다. 강의 논리 점검·상담 케이스 함께 뜯기 |
| **리서치** | `/research` | 커리어 주제를 나리님님 대신 조사해, 나리님님 프레임과 말투로 브리핑합니다 |
| **대리응답** | `/answer` | 코치이(내담자) 질문에 나리님님이라면 할 답을 합니다 — 방금 보신 것처럼 |

세 모드 모두 하나의 심장을 공유해요. 바로 **나리님님의 지식 베이스** — PPT·영상·상담 경험을 모아둔 `knowledge/` 폴더입니다. 코치의 모든 답은 여기서 나오고, 여기에 없으면 "제 자료엔 없어요"라고 정직하게 말합니다.

## 오늘 4시간의 흐름

| 단계 | 무엇을 하나 | 산출물 |
|---|---|---|
| **Part 1 — 코치 정체성** | 코치 작명 + 도메인·화법·고유 관점 인터뷰로 페르소나 초안 작성 | 코치 페르소나<br>`.claude/agents/코치.md` |
| **Part 2 — 지식 베이스 구축** | 내 PPT·영상·상담을 불러와 구조화 → **코치가 내 자료를 인용해 첫 답변** | `knowledge/` 지식 베이스<br>+ 첫 그라운딩 답변 |
| **Part 3 — 3모드 완성·일상화** | 대리응답·토론·리서치 3모드 + 근거 검수 + 일상 습관 | 운영 커맨드 3종<br>`/answer` `/spar` `/research` |

**Part 2**가 오늘의 심장이에요. 흩어져 있던 나리님님 자료가 코치의 지식이 되어, 코치가 처음으로 "나리님님 목소리"로 답하는 순간을 만납니다.

각 챕터는 **[개념] → [실습] → [체크포인트]** 패턴으로 진행됩니다. 실습은 슬래시 커맨드로 코치를 직접 움직여 보는 시간이에요.

## 4시간 뒤의 약속

방금 보신 답변은 **강사(이수연)의 코치가 강사 자료를 인용해** 답한 거예요.

오늘 워크숍이 끝나면, **나리님님의 코치가 나리님님 자료로 똑같이 답합니다.** 강사 슬라이드 자리에 나리님님의 강의 슬라이드가, "AX 전환 실전" 자리에 나리님님의 강의명이 들어갑니다. 그 순간을 Part 2 끝(2-3)에서 직접 만나실 거예요.
